# 场外基金卖出决策

对 $ARGUMENTS 检查已持仓基金是否需要卖出/减仓。支持参数 `--my-holdings` 自动读取自身持仓，或传入基金代码列表手动指定。

**依赖**：`fund-monitor` 前置信号预警池（经理更换/规模预警/费率/负面/范围变更）| `jd_finance_api.py` 自身持仓+大佬持仓 | `fund-checklist` 买入记录 | `fund-penetration` 穿透快照

## 执行流程

**四级信号按变化频率分频执行**：每日只检查 🔴，每周检查 🟡🟢，每月检查 🔵。统一入口 `fund-sell`，内部按频率分步。

| 频率 | 检查信号 | 原因 |
|:----:|---------|------|
| **每日** | 🔴 硬性卖出 | 经理换人、规模跌穿、暂停赎回——这些今天可能发生 |
| **每周** | 🟡 减仓观察 + 🟢 止盈 | 跑输同类需观察趋势、估值不会一周翻倍 |
| **每月** | 🔵 组合管理 | 仓位占比和持仓重叠度变化慢 |

### 步骤 A（每日）：读取输入 + 检查硬性卖出信号（🔴）

#### A1：读取输入——自身持仓 + 前置预警池

自身持仓：
```bash
python3 tools/jd_finance_api.py --my-holdings
```
返回：基金代码、持有金额、收益率、首次买入日期（如API不返回持有时间则从买入记录推算）

前置预警池：读取 `data/pre_alert_cache.json`（由 fund-monitor 每日写入）
```json
{
  "red": [
    {"fund_code": "006105", "reason": "基金经理更换", "detected_at": "2026-06-27"}
  ],
  "yellow": [
    {"fund_code": "000216", "reason": "规模异常暴增", "detected_at": "2026-06-27"}
  ]
}
```

大佬持仓 daily diff：读取 `data/holdings_diff_cache.json`

**输入处理**：
- `fund-sell --my-holdings` → 自动拉取全部持仓
- `fund-sell 006105, 000216` → 只分析指定基金
- 无参数 → 默认 `--my-holdings`

#### A2：检查硬性卖出信号（🔴）

每日必做。**任一条件触发即执行**，不进入后续步骤。

| 条件 | 数据来源 | 说明 |
|------|---------|------|
| `fund-monitor` 红色预警中有该基金 | `data/pre_alert_cache.json` → red[] | ✅ 有 → 清仓 |
| **基金经理变更**（主动型/指数增强） | `getFundManagerListPageInfo` 对比买入经理 | ✅ 换人 → 清仓（纯被动指数不触发） |
| **基金规模<5000万** | `getFundDetailProfilePageInfo` | ✅ <5000万 → 清仓（纯被动指数放宽） |
| **大佬集体清仓≥3人加权清仓>80%** | `data/holdings_diff_cache.json` → 清仓统计 | ✅ ≥3人加权 → 清仓 |
| **基金公司重大负面** | `fund-monitor` 预警池 | ✅ 严重级 → 清仓 |
| **基金被暂停申购/赎回** | API/公告 | ✅ 暂停 → 清仓 |

**输出格式**：
```markdown
## 🔴 卖出建议：硬性卖出

### {基金名}（{代码}）
- **触发信号**：基金经理更换（原经理张三→现任李四）
- **买入日期**：2026-06-01 | **持有收益率**：+12.5%
- **建议操作**：立即清仓
- **理由**：主动基金核心是经理本人，换人等于换基金。原经理任职5年业绩出色，新任经理任职<1年历史业绩不明。
```

### 步骤 B（每周）：检查减仓信号（🟡）

每周执行一次。**跳过已触发硬性卖出的基金**。每触发一个信号按比例减仓，多个信号同时触发时取**最大减仓比例**（不叠加）。

| 信号 | 检测逻辑 | 减仓比例 | 说明 |
|------|---------|:-------:|------|
| **单一大佬清仓** | `--holdings` 检测某个大佬从持有→0 | **1/3** | 单一大佬可能因资金需求卖出，勿过度反应 |
| **大佬集体减仓（≥3人各减>30%）** | 多人同时大幅减仓，无买入抵消 | **1/3** | 有老板B同期加仓→不触发 |
| **风格漂移** | 净值与基准的滚动相关性偏离>阈值 | **1/3** | 需日频净值数据，API待验证 |
| **穿透股进入悲观情景** | 三情景估值悲观概率>50% | **1/2** | 核心重仓股估值恶化 |
| **持续跑输同类** | 连续6个月同类排名后25% | **1/3** | 减仓后再观察3个月，继续跑输再减1/3 |
| **基金规模异常暴增** | 季度环比+200%以上 | **1/3** | 仅主动基金；被动基金不触发 |
| **基金投资范围变更** | 从股票型改为混合型等 | **1/3** | 流程虽长但值得警惕 |
| **费率突然上调** | 管理费超出原费率 | **1/3** | 直接侵蚀收益 |
| **基金限购政策突变** | 日限额大幅降低 | **1/3** | 影响大额资金再平衡 |
| **基金公司投研团队动荡** | 核心研究员集体离职 | **1/3** | 影响后续研究支撑 |
| **巨额赎回导致净值异常波动** | 单日赎回>10%规模 | **1/3** | 可能被迫卖出优质资产 |
| **穿透快照过期** | 距离上次穿透重估>90天 | — | **触发 `fund-penetration` 重新计算三情景估值**（见下方说明） |

**大佬买入抵消逻辑**：
```
如果大佬A清仓某基金，但大佬B同期加仓同一基金 → 信号矛盾
  输出：标记"分歧观察"，不触发减仓，等待后续明确信号
```

**穿透快照过期处理**：如果某基金距离上次穿透分析超过90天，**先触发穿透重估**，再用新的悲观概率判断是否需要减仓。穿透重估调用：
```bash
python3 tools/jd_finance_api.py --fund-holdings {代码}
python3 tools/jd_finance_api.py --fund-penetration {代码}
```

**输出格式**：
```markdown
## 🟡 卖出建议：减仓

### {基金名}（{代码}）
- **触发信号**：穿透股进入悲观情景（PE从30x升至45x，悲观概率65%）
- **当前仓位**：¥8,000（占总仓位12%）
- **建议操作**：减仓1/2 → 减至¥4,000
- **理由**：穿透股英伟达PE突破历史90%分位，三情景悲观概率>50%，建议减半锁定利润。
- **下次检查**：30天后，若穿透股PE回到历史中位数以下可考虑加回
```

### 步骤 C（每周/双周）：检查止盈信号（🟢）

与 🟡 同步执行（每周或每双周）。按基金类型和买入策略读取差异化阈值：

| 基金类型（从买入记录的 YAML frontmatter 读取） | 目标止盈 | 止损线 |
|------------------------|:-------:|:-----:|
| 主动股票型 | +50% | -20% |
| 指数增强型 | +30% | -15% |
| 纯被动指数型 | +20%或PE/PB历史分位 | 不设止损 |
| QDII（主动） | +50% | -25%（汇率+市场双重波动） |
| QDII（被动） | +30%或估值分位 | 不设止损 |
| 定投策略 | 基于持仓成本收益率 | 不设止损 |
| 一次性买入 | 绝对收益+估值分位双重条件 | 按类型表 |
| 分批买入 | 按各批独立收益率算 | 按类型表 |

**额外止盈信号**：

| 信号 | 检测逻辑 | 行动 |
|------|---------|------|
| **穿透股进入极度高估** | PE>历史90%分位（仅穿透到单只核心股时适用） | 止盈1/2 |
| **持仓时间+收益率双重确认** | 持仓>1年 且 年化收益率>目标年化 | 止盈1/3 |

**输出格式**：
```markdown
## 🟢 卖出建议：止盈

### {基金名}（{代码}）
- **触发信号**：达到目标收益率（买入时设定+30%，当前+35.2%）
- **基金类型**：指数增强型
- **当前金额**：¥6,750（原投入¥5,000）
- **建议操作**：止盈1/2 → 赎回¥3,375
- **理由**：持仓7个月已达到指增型目标+30%，建议锁定一半利润。
- **剩余仓位继续持有**：¥3,375继续观察，若继续上涨可在+50%时再止盈剩余。
```

### 步骤 D（每月）：检查组合管理信号（🔵）

| 信号 | 检测逻辑 | 行动 |
|------|---------|------|
| **单只主动基金仓位>15%** | 自身持仓中单只占比>15% | 减仓至10% |
| **单只被动指数基金仓位>20%** | 同上 | 减仓至15% |
| **跨基金持仓重叠度>50%** | 多只基金前5重仓股Jaccard相似度>50% | 减仓重复度高的那只 |
| **更好的替代出现** | `fund-monitor` 新共识信号评分高于当前持仓+调仓成本计算 | 调仓成本≤超额收益30%时执行 |

**调仓成本计算规则**：
```
如果 赎回费 + 申购费 > 预期超额收益 × 30%
  则：不调仓
  否则：执行调仓
```

### 大佬卖出信号加权计算（每日/每周均需读取）

`fund-sell` 不直接拉取大佬持仓，而是读取 `data/holdings_diff_cache.json`（由 fund-monitor 每日维护）。

**缓存格式**：
```json
{
  "date": "2026-06-28",
  "funds": {
    "006105": {
      "code": "006105",
      "name": "XXX基金",
      "clear_total": 0,
      "reduce_30pct_total": 0,
      "clear_details": [
        {"uid": "jimu_user_info-14345330", "name": "Z先生养基", "hold_days": 540, "hold_weight": 2.0},
        {"uid": "jimu_user_info-16020895", "name": "王晴阳", "hold_days": 320, "hold_weight": 1.0}
      ],
      "weighted_clear": 3.0,
      "weighted_reduce_30pct": 1.0,
      "conflict": {"加仓_ids": [], "add_ids": []},
      "verdict": "red_clear"
    }
  }
}
```

**加权计算**：
```
清仓加权人数 = Σ(每位清仓大佬的权重)
  持有>2年 → 权重2.0
  6个月~2年 → 权重1.0
  <6个月 → 权重0.5

加权≥3 → 🔴 红色预警（硬性卖出触发）
加权≥2 → 🟡 黄色预警（标记观察）
减仓>50%加权人数≥2 → 🟡 黄色预警
```

## 与 fund-monitor 的分工

| 分工 | fund-monitor | fund-sell |
|------|-------------|-----------|
| **扫描范围** | 全市场（持仓+候选） | 只检查已持仓基金 |
| **前置信号** | 经理更换/规模预警/费率/负面/范围变更 → 写入预警池 | 读取预警池 |
| **深度信号** | — | 风格漂移、穿透重估、跑输同类、止盈、组合管理 |
| **大佬持仓** | 每日拉取全量+增量diff | 读取 diff 缓存 |
| **预警桶分级** | 🔴红色/🟡黄色/👁观察 | 根据预警级别+自身持仓综合判断 |
| **输出** | 预警信号（共享） | 具体卖出建议（清仓/减仓X%/持有） |

## 注意事项

- **执行频率**：🔴 每日（与 fund-monitor 同步触发）→ 🟡🟢 每周/双周 → 🔵 每月。不要在错误频率上检查错误信号
- **执行顺序**：每日只跑步骤 A（🔴），周一/每周触发跑步骤 B+C（🟡🟢），月初跑步骤 D（🔵）
- **优先级**：🔴 > 🟡 > 🟢 > 🔵（先保本，再止盈，最后再平衡）
- **多信号不叠加**：多个减仓信号同时触发时，取**最大减仓比例**，不叠加（如"持续跑输"减1/3 + "规模暴增"减1/3 = 只减1/3，不是减2/3）
- **穿透过期的触发**：距离上次穿透>90天的基金，每周检查时先触发 `fund-penetration` 重算，再用新估值判断减仓
- 大佬卖出信号存在局限性：大佬可能因资金需求（买房、缴税）卖出，与基金质量无关。单一大佬卖出权重不宜过高
- 止盈止损线从买入记录的 YAML frontmatter 读取，不自作聪明改阈值
- 纯被动指数型不设止损线（指数不会死），也不因经理变更触发卖出
- 保存大佬持仓缓存历史（最近30天），用于回溯"什么时候开始减仓的"
