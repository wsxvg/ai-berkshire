# 场外基金买入前 Checklist

对 $ARGUMENTS 执行场外基金买入前六关 Checklist 分析。

**支持输入格式**：单个或多个基金代码，用逗号/顿号/空格分隔。例如：`006105, 000216` 或 `110011 260108`

**数据来源**：全部从京东金融API实时获取（tools/jd_finance_api.py），费率、T+N、限额等数值绝不硬编码。

## 执行流程

### 第一步：解析输入，识别所有待分析基金

从 $ARGUMENTS 中解析出所有基金代码。对每只基金：
- 调用 `python3 tools/jd_finance_api.py --fund-profile {代码}` 获取基金档案
- 调用 `python3 tools/jd_finance_api.py --trade-rules {代码}` 获取交易规则
- 调用 `python3 tools/jd_finance_api.py --fund-holdings {代码}` 获取持仓分布
- 调用 `python3 tools/jd_finance_api.py --fund-perf {代码}` 获取业绩排名
- 调用 `python3 tools/jd_finance_api.py --fund-manager {代码}` 获取基金经理

如果包含 `--offline` 参数，所有API调用自动使用本地缓存。

### 第二步：硬过滤前置筛（窄口前置）

**目的**：在进入六关之前快速排除明显不合格的基金，减少无效分析。从fund-monitor输出的50-80只候选基金中，快速过滤到10-15只。

**硬过滤规则**（所有条件必须同时满足）：

| 条件 | 阈值 | 数据来源 |
|------|------|---------|
| 共识信号强度 | ≥⭐⭐（≥5人持有） | fund-monitor输出 |
| 基金规模 | ≥2亿 | `getFundDetailProfilePageInfo` |
| 近1年收益 | ≥同类均值-10%（允许适度落后） | `getFundHistoryPerformancePageInfo` |
| 成立年限 | ≥1年 | `getFundDetailProfilePageInfo` |
| 基金经理任职 | ≥6个月 | `getFundManagerListPageInfo` |

**通过硬过滤的基金才进入分析。被过滤掉的基金记录原因即可，不展开分析。**

### 第三步：基金类型识别与分支选择

**目的**：不同类型的基金适用不同的评估框架。主动基金看经理，指数基金看跟踪误差，必须走不同路径。

从 `getFundDetailProfilePageInfo` 获取基金类型字段，按以下规则分类：

| 基金类型 | 识别依据 | 走哪条分支 |
|---------|---------|-----------|
| 主动股票型 / 偏股混合型 | 类型字段含"股票型"+"非指数" | **完整六关**（含基金经理评估） |
| 纯被动指数型 / ETF联接 | 类型字段含"指数型"+"被动"或"ETF联接" | **极简三关**（费率+跟踪误差+指数代表性，跳过经理关/历史业绩关/聪明钱关） |
| 指数增强型 | 类型字段含"指数型"+"增强" | **增强五关**（第三关改为跟踪误差+超额收益评估） |
| QDII（主动） | 类型字段含"QDII"+"股票型/混合型" | **完整六关**，但穿透时切换对应市场估值体系（美股用纳斯达克历史分位，港股用恒生历史分位） |
| QDII（被动） | 类型字段含"QDII"+"指数" | **极简三关**（费率+跟踪误差+指数代表性），穿透时切换对应市场估值体系 |
| 行业主题型 | 投资策略/名称含行业关键词（如"医疗""新能源"） | **完整六关**，但第一关（能力圈）调整为行业景气度判断 |
| FOF / 养老目标 | 类型字段含"FOF"或"养老" | **暂不支持**，穿透需两层展开。在输出报告中明确标注 "FOF类型：当前版本暂不支持两层穿透分析，建议手动关注"，不静默跳过 |

**分支说明**：

- **完整六关**：走标准的六关流程（含基金经理评估），每条评分规则不变
- **极简三关**：只有三关评分维度——**费率**（成本低）、**跟踪误差**（跟紧指数）、**指数代表性**（这只指数是否代表你想要的暴露）。跳过经理关、历史业绩关、聪明钱验证关。不评分规模（硬过滤已卡≥2亿）。
- **增强五关**：走标准六关，但第三关替换为"跟踪误差+信息比率+超额收益稳定性"，不强求经理任职年限

### 第四步：逐基金执行 Checklist（分支）

按第三步识别的基金类型走对应分支：
- **主动型 / 指数增强型 / 行业主题型 / QDII主动** → 走完整六关（第一关至第六关），第三关按A/B分支
- **纯被动指数型 / ETF联接 / QDII被动** → 走极简三关（指数代表性 + 跟踪误差 + 费率），跳过第三至第六关

---

> 纯被动指数型的专属三关评分标准：
> 
> **第一关（指数代表性）**：
> - ★★★★★ 指数精准覆盖目标暴露（如投纳斯达克100→选纳斯达克100ETF）
> - ★★★★☆ 指数相关但并非最直接（如投A股大盘→选中证100而非沪深300）
> - ★★★☆☆ 指数范围过宽或过窄
> - ★★☆☆☆ 指数与目标暴露有明显偏离
> - ★☆☆☆☆ 不理解指数编制规则，或选错指数
> 
> **第二关（跟踪误差）**：
> - ★★★★★ 年化跟踪误差<0.5%，规模大流动性好
> - ★★★★☆ 年化跟踪误差0.5%-1%
> - ★★★☆☆ 年化跟踪误差1%-2%，基本可接受
> - ★★☆☆☆ 年化跟踪误差>2%，偏离严重
> - ★☆☆☆☆ 大幅偏离指数，存在复制方法问题
> 
> **第三关（费率）**：复用第四关评分标准，见下方费率关。
> 
> 以下六关详细标准仅适用于主动型/指数增强型/行业主题型/QDII主动。

---

#### 第一关：我能理解这只基金吗（能力圈）

必须回答：
- [ ] 能说清楚这只基金的投资策略吗？（主动管理/被动跟踪/量化/增强）
- [ ] 跟踪什么指数或投资什么领域？
- [ ] 基金类型是什么？（股票型/债券型/混合型/QDII/货基/FOF）
- [ ] 这只基金适合什么市场环境？

**评分标准**（★1-5）：
- ★★★★★：策略极其简单透明，如纯指数基金（沪深300ETF联接）
- ★★★★☆：策略清晰但需要一定理解，如行业主题基金
- ★★★☆☆：策略可理解但有复杂衍生品或杠杆
- ★★☆☆☆：策略复杂或多层嵌套（如FOF中的FOF）
- ★☆☆☆☆：完全不理解投资逻辑

**硬性否决**：如果连基金投什么都说不清，直接标记为"不在能力圈，不做分析"。

> ⚠️ **行业主题型基金**：除能力圈评估外，需额外判断行业景气度阶段（导入期/成长期/成熟期/衰退期），并将景气度判断融入第一关评分。

**数据来源**：`getFundDetailProfilePageInfo` → 投资目标、投资策略

---

#### 第二关：好基金吗（历史业绩与质量）

用数据说话：

| 指标 | 数值 | 参考标准 | 判断 |
|------|------|---------|------|
| 近1年收益率 | | 同类排名前50% | |
| 近3年收益率 | | 持续跑赢同类均值 | |
| 近5年收益率 | | 长期稳定 | |
| 最大回撤 | | <同类均值为佳 | |
| 最大回撤修复时间 | | 同类均值以内为佳 | |
| 夏普比率 | | >1为佳 | |
| 索提诺比率 | | >0.8为佳，越高越好 | |
| 机构持有人比例 | | >20%加分，<5%警惕 | |
| 同类排名稳定性 | | 连续多期前50% | |

**说明**：
- **最大回撤修复时间**：从净值最低点恢复到前高的月数。修复时间超过同类均值1.5倍 → 说明净值跌下去很难涨回来，往往是结构性问题而非市场波动。
- **索提诺比率**：夏普比率的改良版，只惩罚下行波动。比率高说明"上涨稳、跌得少"。两只基金夏普比率接近时，索提诺高的那只持有体验更好。
- **机构持有人比例**：机构资金占比是"聪明钱验证"的独立维度（与第六关的大佬持仓互补）。机构持有人比例高意味着经过专业买方的筛选。结合半年度/年度报告获取。
  - >20% → 加分
  - <5% 且 成立≥2年 → 警惕（新发基金成立<2年机构比例天然低，豁免此规则）

**硬性否决**：
- 近3年业绩持续垫底（后25%）
- 最大回撤远超同类（>1.5倍同类均值）
- 最大回撤修复时间 > 同类均值2倍（跌了涨不回来）
- 基金规模 < 2亿（有清盘风险）

**数据来源**：`getFundHistoryPerformancePageInfo` → 多周期业绩+排名；持有人结构从基金中期/年度报告获取

---

#### 第三关：管理层评估（按基金类型分支）

> ⚠️ 本关根据基金类型走不同分支：
> - **主动股票型 / 偏股混合型 / 行业主题型 / QDII主动** → 走 **A分支：基金经理评估**
> - **指数增强型** → 走 **B分支：跟踪误差+超额收益**
> - **纯被动指数型 / ETF联接 / QDII被动** → **本关跳过**（计为N/A，不参与加权）

---

##### A分支（主动型）：基金经理靠谱吗

**评分标准**（★1-5）：
- ★★★★★：任职>5年，业绩持续优秀，管理规模适中
- ★★★★☆：任职3-5年，业绩良好，风格稳定
- ★★★☆☆：任职1-3年，业绩尚可，需继续观察
- ★★☆☆☆：任职<1年，或频繁更换经理
- ★☆☆☆☆：经理有重大负面记录

**从API获取的雷达评分**（五个维度）：
- 收益能力
- 选股能力
- 避险能力
- 机会把握
- 投资经验

**基金经理能力系数计算**：
```
系数 = (收益能力×0.3 + 选股能力×0.3 + 避险能力×0.2 + 机会把握×0.2) / 5
修正：任职>5年 → ×1.1，<2年 → ×0.8
最终范围：0.5 ~ 1.5
```

**数据来源**：`getFundManagerListPageInfo` + `getFundManagerDetailPageInfo`

---

##### B分支（指数增强型）：跟踪误差与超额收益

被动指数不需要评估经理，但指数增强型需要评估"能不能跑赢指数"。

| 指标 | 参考标准 | 判断 |
|------|---------|------|
| 跟踪误差（年化） | <2%为佳 | 越小代表紧密跟踪指数 |
| 信息比率 | >0.5为佳 | 超额收益/跟踪误差，越高越好 |
| 近1年超额收益 | >0为达标，>2%为优秀 | 战胜指数的幅度 |
| 近3年超额收益稳定性 | 连续3年正超额 | 风格一致性的关键证据 |

**评分标准**（★1-5）：
- ★★★★★：信息比率>1，连续3年正超额
- ★★★★☆：信息比率>0.5，近2年正超额
- ★★★☆☆：超额收益不稳定，时正时负
- ★★☆☆☆：长期跑输指数，费用白付
- ★☆☆☆☆：跟踪误差大+超额为负

**数据来源**：`getFundHistoryPerformancePageInfo` → 超额收益、跟踪误差

---

#### 第四关：费率合理吗（成本）← 股票没有这一关

**所有费率必须从 `getFundTradeRulesPageInfo` 实时获取，绝不能硬编码。**

| 费用项 | 本基金 | 同类均值 | 判断 |
|--------|--------|---------|------|
| 申购费率（折扣后） | | 通常0.1%-0.15% | |
| 管理费（%/年） | | 股票型1.5%，债基0.3%-0.7% | |
| 托管费（%/年） | | 通常0.1%-0.25% | |
| 销售服务费（%/年） | | C类有，A类无 | |
| 赎回费率（<7天） | | 1.5%（证监会强制） | |
| 赎回费率（≥365天） | | 通常0% | |

**实际持有成本计算**：
```
持有1年总成本 = 申购费 + 管理费 + 托管费 + 销售服务费
持有3年总成本 = 申购费 + 3×(管理费+托管费+销售服务费)
```

**A类 vs C类选择**：
- 持有 < 1年：C类通常更划算（无申购费但有销售服务费，销售服务费通常0.4%-0.8%/年）
- 持有 > 1年：A类通常更划算（一次性申购费但无销售服务费，申购费折扣后通常0.1%-0.15%）

**自动推荐规则**（融入最终决策输出）：
- 计划持有>1年 → 推荐A类份额
- 计划持有<1年或不确定 → 推荐C类份额
- C类持有超过2年总成本可能反超A类，需在定投建议中注明

**硬性否决**：
- 管理费 > 2%/年（极少数例外）
- 综合持有成本远超同类（>1.5倍）

**数据来源**：`getFundTradeRulesPageInfo` → 完整费率分档

---

#### 第五关：流动性够吗（退出成本）← 股票没有这一关

| 项目 | 数值 | 判断 |
|------|------|------|
| T+N确认天数 | | QDII可能T+7~T+10 |
| 买入截止时间 | | 15:00前当日净值 |
| 日累计限额 | | 限额太低影响大额买入 |
| 基金规模 | | <2亿有清盘风险 |
| 最低申购金额 | | 通常1元起 |

**硬性否决**：
- T+N > 10天（极端情况需评估）
- 日限额 < 计划买入金额
- 基金规模 < 2亿

**数据来源**：`getFundTradeRulesPageInfo` → T+N、限额、截止时间；`getFundDetailProfilePageInfo` → 规模

---

#### 第六关：大佬在买吗（聪明钱验证）

**双重验证**：大佬持仓 + 独立评估

1. 调用 `python3 tools/jd_finance_api.py --holdings jimu_user_info-{uid}` 获取关注人持仓
2. 交叉验证信号强度：

| 信号类型 | 判断 | 说明 |
|---------|------|------|
| 多人同时买入 | ⭐⭐⭐ | 3人以上同时新进 |
| 大额买入 | ⭐⭐⭐ | 单笔>1万元 |
| 持续加仓 | ⭐⭐ | 连续多月增加 |
| 持有不动 | ⭐ | 长期持有未变动 |
| 减持/清仓 | ⚠️ | 大佬在卖，需关注 |

**核心原则**：大佬买 ≠ 你该买。大佬可能亏钱，大佬的仓位/期限/目标可能与你不同。大佬信号是**参考**，不是**决策依据**。

**数据来源**：`queryUserFundHoldingInfo` → 持仓；`getPageMutilData?pageId=11568` → 交易记录

### 第五步：综合评分与决策

#### 评分汇总表

> ⚠️ 评分模板按基金类型调整：
> - **主动型** → 六关全评，第三关走A分支（基金经理）
> - **指数增强型** → 六关全评，第三关走B分支（跟踪误差+超额收益）
> - **纯被动指数型 / ETF联接** → 走极简三关（费率+跟踪误差+指数代表性）。第六关（聪明钱验证）对纯被动基金意义弱——大佬买沪深300ETF通常是为了Beta配置，不是因为"这只ETF的基金经理很牛"。权重转移给第一关（指数代表性）。

**主动型 & 指数增强型评分表**：

| 关卡 | 评分(★) | 关键发现 | 是否否决 |
|------|---------|---------|---------|
| 第一关：理解 | /5 | | |
| 第二关：质量 | /5 | | |
| 第三关：【经理/跟踪误差】 | /5 | | |
| 第四关：成本 | /5 | | |
| 第五关：流动性 | /5 | | |
| 第六关：聪明钱 | /5 | | |

**综合评分（主动/指数增强）** = (第一关×0.15 + 第二关×0.25 + 第三关×0.2 + 第四关×0.15 + 第五关×0.1 + 第六关×0.15)

**纯被动指数型评分表**（极简三关）：

| 关卡 | 评分(★) | 关键发现 | 是否否决 |
|------|---------|---------|---------|
| 第一关：指数代表性 | /5 | 这只指数是否代表你想要的暴露？ | |
| 第二关：跟踪误差 | /5 | 是否紧密跟踪指数 | |
| 第三关：费率 | /5 | 综合持有成本是否足够低 | |

**综合评分（纯被动指数）** = (指数代表性×0.40 + 跟踪误差×0.30 + 费率×0.30)
> 纯被动基金的核心差异在"选哪个指数"，而非"谁在买"或"历史业绩"。指数代表性权重最高。

#### 决策矩阵

| 综合评分 | 建议 |
|---------|------|
| ≥4.0 | 值得深入研究，可考虑建仓 |
| 3.5-4.0 | 基本面合格，需择时 |
| 3.0-3.5 | 有明显短板，谨慎观望 |
| <3.0 | 不建议买入 |

#### 穿透分析触发

如果综合评分 ≥ 3.5，自动触发 `fund-penetration` 对底层重仓股进行穿透评估：

```bash
python3 tools/jd_finance_api.py --fund-holdings {基金代码}
# 对每只重仓股 → 调用 /investment-checklist
```

#### 五维评分参考（如数据可用）

如果 `data/auto/status.json` 中包含 `fund_scores` 字段（由 auto-pipeline.py Step 3b 自动写入），可以在输出报告中附加评分引擎结果供参考：

```python
# 读取评分数据
import json
scores = json.load(open("data/auto/status.json"))["fund_scores"]
# scores[code] = {"total": 3.42, "verdict": "watch"}
```

**评分引擎信息**（tools/fund_scorer.py）：
- 五维：质量分×0.25 + 成本分×0.20 + 经理分×0.20 + 动量分×0.15 + 聪明钱×0.20
- ≥4.0 建议买入，3.3~4.0 值得关注，<3.3 建议跳过
- 评分是确定性数值计算（非LLM），LLM只负责解读
- 评分引擎有反证规则（规模<5000万→否决、经理换人→否决、数据过时→降权）
- 数据新鲜度标记：超90天数据自动降权10%

**评分 vs 手动六关的关系**：
- 六关评分更精细（包含流动性、理解度等主观维度）——适合人工深度分析
- 五维评分更稳定（纯数值、可回溯、可对比）——适合自动化扫描
- 两者一致时→信心增强。不一致时→说明基金在流动性/理解度等维度存在特殊问题

## 第七步：买入记录区（买入时填写）

**每次买入决策通过后**，在 checklist 报告中追加记录完整的买入元数据。**记录使用 YAML frontmatter 格式**（文件顶部），以便 `fund-sell` 用机器方式解析读取，避免正则从自然语言段落中提取的不稳定性。

```
---
buy_date: "2026-06-28"
buy_price: 1.234
initial_amount: 5000
fund_type: "active"          # active | index_enhanced | passive_index | qdii_active | qdii_passive
buy_strategy: "lump_sum"     # lump_sum | dca | batch
target_profit: 0.50          # +50%（主动型）/ +30%（指增）/ +0.20或估值分位（被动）
stop_loss: -0.20             # -20%（主动型）/ -15%（指增）/ null（被动不设）
buy_entry_penetration:
  pe: 25
  pb: 4.5
  scenario_optimistic: 0.30
  scenario_base: 0.50
  scenario_pessimistic: 0.20
checklist_score: 4.2
checklist_rating: "pass"     # pass | conditional | fail
reasons:
  - "基金经理张三任职5年业绩持续同类前20%"
  - "穿透股英伟达PE=25x处于历史中位数偏低"
  - "大佬Z先生和王晴阳同时重仓"
falsify_conditions:
  - "manager_change"         # 经理换人即证伪
  - "pe_exceeds_80"          # 穿透股PE>80x即证伪
  - "big_player_clear_3plus" # 大佬集体清仓≥3人即证伪
---
```

**字段说明**：

| 字段 | 类型 | 可选值 | 说明 |
|------|------|-------|------|
| `fund_type` | string | `active` / `index_enhanced` / `passive_index` / `qdii_active` / `qdii_passive` | 决定止盈止损阈值和卖出策略 |
| `buy_strategy` | string | `lump_sum` / `dca` / `batch` | 一次性 / 定投 / 分批，影响止盈计算方式 |
| `target_profit` | float | 0.10–1.00 | 目标收益，sell 读取此值与当前收益率对比 |
| `stop_loss` | float or null | 负数或 null | 止损线，null = 不设（被动指数） |
| `falsify_conditions` | string[] | 见下方 | 证伪条件列表，sell 逐条检查 |
| `buy_entry_penetration` | dict | — | 买入时的穿透快照，sell 对比当前估值 |

**自动生成的 `falsify_conditions` 取值**：

| 评分时的发现 | 自动生成的值 | sell 检查逻辑 |
|------------|-------------|--------------|
| 经理任职<3年 | `"manager_change"` | 持仓经理姓名是否变更 |
| 穿透股集中度高 | `"pe_exceeds_80"` 或 `"pb_exceeds_15"` | 当前PE/PB是否超过阈值 |
| 仅1位大佬持仓 | `"sole_big_player_clear"` | 唯一持仓大佬是否清仓 |
| 规模偏低（2-5亿） | `"size_below_150m"` | 规模是否<1.5亿 |
| 指数增强IR<0.5 | `"ir_below_0.3_two_quarters"` | 信息比率连续2季<0.3 |

### 证伪条件自动生成规则

从 checklist 评分过程中自动提取关键风险点：

| 评分时的发现 | 自动生成的证伪条件 | 示例 |
|------------|------------------|------|
| 经理任职<3年 | "经理更换即证伪" | 经理在买入1个月内离职→触发 |
| 穿透股>70%仓位集中于单只股票 | "重仓股PE>90x即证伪" | 英伟达PE从25x涨到90x→触发 |
| 大佬共识仅1人 | "唯一持仓大佬清仓即证伪" | 只有Z先生买，他卖出→触发 |
| 基金规模在2-5亿之间（偏低） | "规模<1.5亿即证伪" | 规模从3亿跌到1.2亿→触发 |
| 指数增强型信息比率<0.5 | "信息比率连续2季度<0.3即证伪" | 超额收益持续恶化→触发 |

**填写说明**：
- 证伪条件由系统自动提取、人工确认后写入
- 条件一般设3-5条，覆盖"什么情况下当初的判断错了"
- 买入记录保存到 `reports/{基金名}/` 同目录下的报告中

## 注意事项

- 所有费率、T+N、限额数据**必须从API实时获取**，绝不能硬编码
- 每只基金的交易规则都不同，必须逐基金获取
- QDII基金的T+N天数可能很长（T+7~T+10），需特别关注
- **基金类型必须在硬过滤后立即识别**，根据类型走不同分支。纯被动指数基金跳过第三关
- 基金经理能力系数从API雷达评分实时计算，不使用固定值（仅主动型走A分支时使用）
- QDII穿透到美股时，估值中枢按纳斯达克历史PE分位判断，不套用A股标准；穿透到港股时按恒生历史PE分位判断
- 报告语言中文，风格直接犀利，不说废话
