# 策略 D 季度检视

在季度末（3/6/9/12月）使用，基于策略 D 框架给出持仓检视和调仓建议。

## 触发方式

"季度检视" / "策略 D 检视" / "调仓分析" / "fund-strategy-d-review"

## 执行流程

### 第一步：确认数据新鲜度

```bash
python scripts/auto-pipeline.py --offline
```

读取 `data/auto/status.json`，确认：
- `date` 是否是近3天内的数据
- `cookie_ok` 是否为 true
- `holdings_ok` 是否为 true

如果数据不是近3天的，提示用户先运行 `python scripts/auto-pipeline.py`。

### 第二步：读取策略配置和当前持仓

读取 `data/strategy_D_config.json` — 策略参数和硬约束

读取 `data/holdings_snapshot.json` — 大佬最新持仓
读取 `data/auto/status.json` — 信号汇总、用户排名
读取 `reports/auto/latest.md` — 今日信号报告

### 第三步：市场状态判断

```bash
python -c "from backtest.engine.backtest import detect_market_state; import json; fc=json.load(open('backtest/data/fund_charts.json')); print(detect_market_state('$(date +%Y-%m-%d)', fc))"
```

判断当前市场处于：牛市 / 震荡 / 熊市

### 第四步：分析每只持仓基金

对当前持仓的每只基金（从 holdings_snapshot 中提取用户的持仓），调用以下分析：

1. **基金评分分析** — 调用 `fund-analyze` 的底层逻辑：
   - 运行 `python tools/fund_scorer.py --score CODE` 获取5维评分
   - 解读评分变化趋势（相比上次检视）
   
2. **持仓穿透分析** — 调用 `fund-penetration` 的底层逻辑：
   - 股票/债券/现金占比
   - 前10大持仓集中度
   - 行业分布

3. **经理稳定性** — 检查基金经理是否变更：
   - 对比上次检视时的经理信息
   - 如果变更，标记 ⚠️

4. **规模变化** — 基金规模是否异常变动：
   - 规模从 50亿→120亿 可能影响超额收益
   - 规模从 5亿→2亿 可能有清盘风险

### 第五步：大佬动态分析

从 `data/holdings_snapshot.json` 和 `data/auto/status.json` 分析：

1. **大佬集体买入** — 哪些基金被多个大佬同时买入（建仓/加仓）
2. **大佬集体卖出** — 哪些基金被多个大佬清仓/减仓
3. **新面孔** — 大佬新买入的基金（之前没持有过）
4. **共识变化** — 相比上次检视，共识是增强还是减弱

### 第六步：核心-卫星调仓决策

**核心仓位（60%）决策：**
- 当前核心基金评分是否仍然靠前（排名前30%）？
- 如果是 → 继续持有 ✅
- 如果否 → 寻找替代基金 🔄
- 核心基金出现经理变更/规模异常 → 降级为卫星 🔄

**卫星仓位（40%）决策：**
- 根据市场状态调整风格：
  - 牛市 → 增配科技/成长
  - 震荡 → 均衡配置
  - 熊市 → 转防御/现金
- 卫星基金表现不佳（连续2个月跑输同类中位数）→ 换掉 🔄

**硬约束检查：**
- 单只基金是否超过30%上限？
- 年交易次数是否超过6次？
- 30天持有期是否满足？

### 第七步：输出季度检视报告

```
## 📋 策略 D 季度检视报告（2026-QX）

### 市场状态
当前市场：**牛市/震荡/熊市**
（基于CSI300近60日走势判断）

### 📊 当前持仓分析

| 基金 | 类型 | 评分 | 持仓占比 | 建议 |
|------|------|------|---------|------|
| XX基金 | 核心 | 4.2/5 | 30% | ✅ 持有 |
| YY基金 | 核心 | 3.8/5 | 25% | ✅ 持有 |
| ZZ基金 | 卫星 | 3.5/5 | 15% | ⚠️ 关注（经理变更）|

### 持仓穿透总览
- 综合股票仓位：XX%
- 综合债券仓位：XX%
- 前10大持仓集中度：XX%
- 板块分布：半导体XX% 科技XX% 消费XX%

### 🏆 大佬动态
- 大佬集体买入：XX基金（3人）、YY基金（2人）
- 大佬集体卖出：ZZ基金（2人清仓）
- 大佬新建仓：AA基金（值得关注）

### 🎯 调仓建议
- 核心仓位：不动/换XXX（理由）
- 卫星仓位：减XXX/加XXX（理由）
- 预计交易次数：X次（剩余额度X次/年）

### ⚠️ 风险提示
- 基金A规模增长过快，可能影响超额收益
- 基金B经理变更，新经理需观察
- 当前权益仓位XX%，若市场转向有下行风险
```

### 第八步：记录检视结果

将本次检视的关键数据保存到 `data/strategy_d_reviews.json`，供下次对比。

## 关键原则

1. **不替用户做决定**：给建议，最终调仓由用户决定
2. **诚实面对不确定性**：信号不明确时说"不确定"
3. **控制交易频率**：季检视最多调2-3只，不频繁换手
4. **关注变化**：重点分析相比上次检视的变化，不是重复描述

## 与其他 Skill 的关系

- `fund-trade`：日常交易建议（轻量级）
- `fund-strategy-d-review`：季度深度检视（本 skill）
- `fund-analyze`、`fund-penetration`、`fund-compare`：底层分析工具
