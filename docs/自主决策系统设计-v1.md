# AI Berkshire 基金自主决策系统设计 v1.2

## 问题定义

**现状**：当前系统主要做两件事——
1. 跟投大佬（看大佬买什么就跟着买）
2. 信号汇总（大佬买入/卖出统计）

**问题**：
- 大佬买入 ≠ 你应该买。大佬的仓位、期限、资金量、风险承受力都与散户不同
- 基金本身的质地（持仓穿透、经理、费率、NAV趋势）没有被量化评估
- 买入和卖出缺乏统一的决策框架，全是经验规则
- `fund-scan` 输出"四大师评价"，但那是在 IDE 里用 LLM 生成的文本评论，不是可量化的评分

**目标**：从"跟投大佬"升级为**基于基金质地的自主决策引擎**——把大佬信号当作一个输入维度，而非决策本身。

---

## 核心设计理念

### 三层信号模型

```
输入层（数据）                   决策层（评分）                  输出层（动作）
───────────────                 ───────────────                ──────────────

  大佬交易流水 ───┐                  ┌── Buy Score ──→ 买入建议
  大佬持仓变动 ───┤                  │
  基金NAV历史  ───┼──→ 五维评分 ──→ ┤── Hold Score ─→ 持有/不动
  持仓穿透估值 ───┤                  │
  基金费率结构 ───┤                  └── Sell Score ─→ 卖出/减仓
  经理雷达评分 ───┘
```

**关键原则**：大佬信号只占一个维度（权重20%），基金基本面占80%。

### 股票分析技术如何落地到基金

`jd_finance_api.py` 已经能获取基金的前十大持仓股票，所以**穿透分析不需要新数据源**。关键是把股票分析框架改造搬过来：

| 股票分析技术 | 基金场景的落地 | 数据来源 |
|-------------|--------------|---------|
| PE/PB 估值分位 → 判断贵不贵 | **穿透到前十大重仓股**，算加权PE分位 | `--fund-holdings` 已经返回 |
| 技术分析（趋势/动量） | **NAV趋势分析**——基金净值本身做20日/60日趋势 | `--fund-chart` 有净值数据 |
| 行业分布/集中度 | **基金持仓行业分布** + 与基准指数的偏离度 | `--fund-holdings` 有行业分配 |
| 多因子评分 | 改造为**基金五因子**（质量/成本/经理/动量/聪明钱） | 已全部有API |
| 回测验证策略 | **基金历史NAV回测**——定投/分批/一次性策略模拟 | 已有净值数据（`--fund-chart`） |

## 一、五维评分模型（核心）

### 评分尺

每个维度采用 0-5 分制，全局统一的评分含义：

| 分数 | 含义 | 示例 |
|------|------|------|
| 5 | 同类前 10%，无显著缺陷 | 收益排名 top 10%，回撤可控，费率最低档 |
| 4 | 同类前 30%，有小缺陷 | 收益排名前 30%，费率略高于最低档 |
| 3 | 同类中位数，可接受 | 收益排名 40-60%，费率行业平均 |
| 2 | 同类后 30%，有显著问题 | 收益排名后 30%，费率偏高，经理经验不足 |
| 1 | 同类后 10%，硬伤 | 收益排名后 10%，费率最高档，经理新上任 |
| 0 | 数据缺失或极差 | 无法获取有效数据，或出现极端负向事件 |

总分 = 各维度评分 × 权重的加权和，保持 0-5 范围。

### 各指标评分映射标准

每个子指标由原始数据（排名分位、费率百分比、金额等）映射到 0-5 分。以下为各维度的映射函数定义，实现时直接照搬。

#### 质量分映射

| 子指标 | 原始数据 | 映射规则 |
|--------|---------|---------|
| 近1年排名 (30%) | rank_pct ∈ [0,1] | 5 × (1 - rank_pct)，即 top 10% → 4.5，top 50% → 2.5。缺数据 → 2.5 |
| 近3年排名 (25%) | rank_pct ∈ [0,1] | 同上 |
| 最大回撤 (20%) | max_drawdown ∈ [0,100] | ≤10% → 5.0, ≤15% → 4.0, ≤20% → 3.0, ≤30% → 2.0, >30% → 1.0。缺数据 → 默认15%→4.0 |
| 规模稳定性 (15%) | — | （需扩展）当前默认 3.0 |
| Sharpe比率 (10%) | sharpe ∈ [-∞,∞] | ≥2.0 → 5.0, ≥1.5 → 4.0~5.0(线性), ≥1.0 → 3.0~4.0, ≥0.5 → 1.5~3.0, >0 → 0~1.5, ≤0 → 0。缺数据 → 0 |

#### 成本分映射

| 子指标 | 原始数据 | 映射规则 |
|--------|---------|---------|
| 管理费+托管费 (35%) | total_annual ∈ [0,∞) | ≤0.2% → 5.0, ≤0.6% → 4.0, ≤1.0% → 3.0, ≤1.5% → 2.0, >1.5% → 1.0 |
| 持有1年总成本 (25%) | total_1y ∈ [0,∞) | ≤0.5% → 5.0, ≤1.0% → 4.0, ≤1.5% → 3.0, ≤2.5% → 2.0, >2.5% → 1.0 |
| 持有3年总成本 (25%) | total_3y = 管理费×3+申购费×折扣 | log映射：5 - ln(total_3y + 0.5)×1.8，指数0.45%→4.5, 主动4.5%→2.0 |
| 赎回费率 (15%) | — | 当前默认 5.0（场外基金持有超1年通常免赎回费） |

#### 经理分映射

| 子指标 | 原始数据 | 映射规则 |
|--------|---------|---------|
| 雷达综合评分 (35%) | radar_score ∈ [0,100] | score/20（归一化到 0-5）。缺数据 → 2.5 |
| 任职年限 (25%) | years | ≥8年 → 5.0, ≥5年 → 4.0, ≥3年 → 3.0, ≥1年 → 2.0, <1年 → 1.0。缺数据 → 3.0 |
| 任职期间年化超额 (20%) | — | （需扩展）当前默认 3.0 |
| 管理规模 (10%) | — | 当前默认 3.0 |
| 管理基金专注度 (10%) | — | 当前默认 3.0 |

#### 动量分映射

| 子指标 | 原始数据 | 映射规则 |
|--------|---------|---------|
| 短期趋势 (25%) | current/MA20 - 1 | ((ratio - 1) × 100 + 2.5)，capped 0-5。即跑赢均线 2.5% → 5.0，跑输 2.5% → 0 |
| 中期趋势 (25%) | MA60 斜率 | slope × 2 + 2.5，capped 0-5。缺60日数据 → 2.5 |
| 回撤恢复 (15%) | (current - peak)/peak | ≥0（创新高）→ 5.0, >-5% → 4.0, >-10% → 3.0, >-20% → 2.0, ≤-20% → 1.0 |
| 动量持续期 (20%) | — | 当前默认 3.0（需扩展） |
| 基准相关性变化 (15%) | — | 当前默认 3.0（需扩展） |

#### 聪明钱分映射

见下文"聪明钱合成公式"。

### 评分维度定义（v1.1 修正）

```
五维评分 = 质量分×0.25 + 成本分×0.20 + 经理分×0.20 + 动量分×0.15 + 聪明钱×0.20

（纯被动指数型：指数代表性×0.30 + 跟踪误差×0.25 + 规模稳定性×0.15 + 成本×0.30，跳过经理和动量）
```

**权重调整理由**（来自 AI 评审反馈）：
- 成本从15%提至20%：场外基金持有周期以月/年计，管理费1.5% vs 指数0.15%的10倍差异在复利下影响巨大
- 动量从20%降至15%：基金净值天然平滑（分散持仓），短期NAV动量意义远小于股票

### 维度1：质量分（Quality）— 权重25%

评估基金本身质地。**所有数据已有**。

| 指标 | 权重 | 数据来源 | 当前状态 |
|------|------|---------|---------|
| 近1年收益率同类排名 | 30% | `--fund-perf` | ✅ 已有 |
| 近3年收益率同类排名 | 25% | `--fund-perf` 跨年数据 | ✅ 已有 |
| 最大回撤 vs 同类均值 | 20% | `--fund-chart` 净值计算 | ✅ 已有（需多期数据） |
| 规模稳定性（半年无剧烈波动） | 15% | `--fund-profile` 规模历史 | ⚠️ 需扩展 |
| Sharpe比率（年化） | 10% | `--fund-chart` NAV/日收益计算 | ✅ v1.2 已实现 |

### 维度2：成本分（Cost）— 权重20%

**基金特有的维度**。买卖股票只有佣金+印花税，基金有申购费/管理费/托管费/赎回费一套体系。持有3年，主动基金4.5% vs 指数0.45%——这是10倍的差异。

申购费折扣率作为可配置参数（`PURCHASE_DISCOUNT`，默认1折 = 0.1），避免渠道差异导致评分失真。持有3年总成本使用对数映射保证区分度：指数总成本0.45%→4.5分，主动总成本4.5%→2.0分。

| 指标 | 权重 | 数据来源 | 当前状态 |
|------|------|---------|---------|
| 管理费+托管费 vs 同类均值 | 35% | `--trade-rules` | ✅ 已有 |
| 持有1年总成本（含申购费折扣） | 25% | `--trade-rules` | ✅ 已有 |
| 持有3年总成本 vs 同类 | 25% | `--trade-rules` 计算 | ✅ 已有 |
| 赎回费率（短期惩罚） | 15% | `--trade-rules` | ✅ 已有 |

### 维度3：经理分（Manager）— 权重20%

**主动基金的核心维度**。

| 指标 | 权重 | 数据来源 | 当前状态 |
|------|------|---------|---------|
| 雷达综合评分（收益+选股+风控+经验） | 35% | `--fund-manager` | ✅ 已有（雷达图五项） |
| 任职年限 | 25% | `--fund-manager` | ✅ 已有 |
| 任职期间年化超额收益 | 20% | 净值 vs 基准计算 | ⚠️ 需从基金信息中提取基准指数 |
| 管理规模（过大可能影响超额） | 10% | `--fund-manager` | ✅ 已有 |
| 历史管理基金数量（专注度） | 10% | `--fund-manager` | ✅ 已有 |

### 维度4：动量分（Momentum）— 权重15%

**从股票技术分析改造而来**。不做K线分析，对基金净值做趋势判断。注意：基金净值天然平滑，动量信号远弱于股票，15%权重合理。

| 指标 | 权重 | 计算方法 | 数据来源 |
|------|------|---------|---------|
| 短期趋势（近20交易日NAV） | 25% | 20日移动平均 vs 当前净值 | `--fund-chart` 净值序列 |
| 中期趋势（近60交易日NAV） | 25% | 60日移动平均斜率 | `--fund-chart` 净值序列 |
| 动量持续期（连续上涨/下跌天数） | 20% | 近N日连续正收益天数 | `--fund-chart` 日收益计算 |
| 回撤恢复阶段（是否在爬坑） | 15% | 距历史高点距离 vs 同类 | `--fund-chart` 计算 |
| 与基准指数相关性变化 | 15% | 60日滚动相关性 vs 基准 | `--fund-chart` + 基准指数数据 |

**为什么需要这个**：你之前在 fund-scan 里发现的"近一周收益为负5.28%、近一个月才3.82%"就是对动量维度的需求——你关心的是"最近趋势怎么样"，不是"长期好不好"。

### 维度5：聪明钱分（Smart Money）— 权重20%

**大佬信号的归宿**——把它变成一个量化的参考维度，而不是决策本身。

| 指标 | 权重 | 计算方法 | 数据来源 |
|------|------|---------|---------|
| 大佬首次建仓（权重×1.5） | 30% | 近3日内首次新进入（非加仓），count×1.5 | ✅ 可从 holdings_diff 区分 |
| 大佬买入共识（加仓，权重×1.0） | 30% | 当日交易流水买入人数 | ✅ `auto-pipeline.py` 已计算 |
| 大佬持仓广度（持有该基金的大佬数） | 25% | `holdings_snapshot` 持有者计数 | ✅ 已有 |
| 穿透股被大佬持有数 | 15% | 前十大重仓股被几位大佬通过其他基金持有 | ⚠️ 需交叉计算 |

**合成公式**：

```
a = count(首次建仓)          # 近3日内首次进入的大佬数
b = count(买入共识)          # 当日交易流水买入人数
c = count(持有该基金的大佬)   # 总持仓广度
d = count(穿透重叠大佬)       # 前十大重仓股被多少大佬持有（Phase 3 实现）

raw_first = min(a, 3) × 1.5 × 1.67     # 首次建仓：最多3人，×1.5权重，归一化到0-5
raw_add   = min(b, 5) × 1.0 × 1.0      # 加仓：最多5人，×1.0权重
raw_breadth = min(c / 3, 1.0) × 5.0    # 广度：每1/3的大佬持有→1分，满5人→5分
raw_penetrate = min(d / 5, 1.0) × 5.0  # 穿透：每1/5覆盖→1分（Phase 3）

smart_money_score = (raw_first × 0.30 + raw_add × 0.30 + raw_breadth × 0.25 + raw_penetrate × 0.15)

# 特殊门槛：没有首次建仓 → 聪明钱分最高不超过3.0
if a == 0:
    smart_money_score = min(smart_money_score, 3.0)
```

**逻辑说明**：
- 首次建仓是聪明钱最强的信号（×1.5），因为没有既得仓位的新进入意味着大佬在"发现"而不是"补仓"
- 加仓次数上限5人（很少有超过5个大佬在同一天加仓同一只基金）
- 广度超过5人不再加分（进一步增加没必要，5人已足够代表共识）
- 没有首次建仓时最高3.0——纯加仓信号不足以推动高分

**关键变化**：以前"3个大佬买入=strong_buy"直接就建议跟。现在"3个大佬买入"只贡献聪明钱维度的一部分（最高5分×20%=1.0总评分），还要看其他四个维度合起来是否≥3.3才建议买入。

## 二、决策流程（新）

### 买入决策

```
┌─────────────────────────────────────────────────────┐
│                  买入触发条件                         │
│  ① 大佬买入共识（trading_records 显示买入）         │
│  ② 大佬新增持仓（holdings_diff 新进入）              │
│  ③ 用户主动指定基金代码                              │
│  ④ 定期再平衡扫描（定时触发）                        │
└────────────────────┬────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────┐
│              五维评分计算                             │
│  质量分(25%) + 成本分(20%) + 经理分(20%)            │
│  + 动量分(15%) + 聪明钱分(20%)                      │
│                                                     │
│  数据类型分支：主动/指增 → 五维全评                    │
│  纯被动指数 → 四维（指数代表性/跟踪误差/规模/成本）   │
└────────────────────┬────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────┐
│           综合决策                                   │
│  ≥4.0  → 强烈建议买入 + 穿透分析触发                  │
│  3.3~4.0 → 可买入，标注风险点                        │
│  ≥4.0（独立发现）→ 可独立买入（即使大佬没买）         │
│  3.0~3.3 → 观察，不入场                              │
│  <3.0  → 不买                                        │
│                                                     │
│  ⚠️ 即使大佬集体买入，如果五维评分<3.0也不买          │
│  ⚠️ 即使大佬不买，如果五维评分≥4.0也可以独立买入      │
└────────────────────┬────────────────────────────────┘
```

**阈值说明**：
- ≥4.0 保留（强烈买入信号应该稀缺）
- 3.3~4.0 放宽（原3.5~4.0），基金触发频率远低于股票，门槛太高=零建议
- <3.3 不买

### 卖出决策

卖出决策复用现有 `fund-sell` 的四级信号框架（🔴 🟡 🟢 🔵），但在"减仓观察"和"止盈"中融入五维评分的变化趋势。

| 信号 | 原有逻辑 | 量化判定条件 | 行动 |
|------|---------|------------|------|
| 🔴 经理换人 | 直接清仓 | — | 清仓（硬性卖出） |
| 🔴 规模<5000万 | 直接清仓 | — | 清仓（硬性卖出） |
| 🔴 大佬集体清仓≥3 | 加权计算 | 重算评分：总分≥4.0且动量分健康 → 可能是系统因素，不操作；否则清仓 | 待评分确认后执行 |
| 🟡 单一大佬清仓 | 减仓1/3 | 总分降≥0.3 **且** 动量分降≥0.5 | 减仓1/3（评分下降确认） |
| 🟡 单一大佬清仓 | 减仓1/3 | 总分降<0.3 **或** 动量分降<0.5 | 只减1/4（可能是大佬个人资金需求） |
| 🟡 多人减仓≥3 | 减仓1/3 | 动量分<3 | 减仓1/3（确认撤退信号） |
| 🟡 多人减仓≥3 | 减仓1/3 | 动量分≥4（评分没降） | 不操作（可能是调仓不是恐慌） |
| 🟢 止盈 | 固定阈值 | 动量分转负（低于上次评分） | 止盈多些（趋势转弱） |
| 🟢 止盈 | 固定阈值 | 动量分仍正 | 止盈少些（趋势仍在） |
| 🔵 仓位过重 | 减仓至阈值 | — | 减仓至阈值（不变） |

### 决策冷却机制

评分每日更新，但场外基金不应日频交易。评分引擎输出 `compare(previous)` 方法控制决策震荡：

| 条件 | 行为 |
|------|------|
| **总分变化 < 0.3 且 所有维度变化 < 0.5** | 触发 `override: "hold"` — 维持上次决策不变 |
| **总分变化 ≥ 0.3 或 任一维度变化 ≥ 0.5** | 正常通过新评分裁决 |
| **买入后不足20个交易日** | 不触发卖出决策（冷却期匹配场外基金赎回周期） |

冷却期和变化阈值由调用方（管道/IDE 流程）执行状态检查，评分引擎只输出 `compare()` 的分析结果。避免"今天建议买入，明天建议观望"的震荡。

### 🔴级信号统一确认规则

所有大佬卖出信号（包括🔴级多人集体清仓）**必须经过评分重算确认**才能执行：

| 原规则 | 新规则 |
|--------|--------|
| 🔴 大佬集体清仓≥3 → 直接清仓 | 🔴 大佬集体清仓≥3 → 重算评分，若总分≥4.0且动量分健康 → 不操作（可能是系统性赎回） |
| 🟡 单一大佬清仓 → 需要评分确认 | 不变 |

理由：多人集体清仓也可能是因为大盘暴跌被迫清仓（产品到期/客户赎回），不应盲目跟出。

## 四、数据时效标记（P0）

每个评分维度必须携带数据新鲜度标记，避免用3个月前过时数据做决策：

```python
@dataclass
class DimensionScore:
    score: float       # 0-5
    weight: float      # 维度权重
    freshness_days: int  # 数据距今天数
    stale: bool        # freshness_days > 90 → True

def apply_stale_penalty(dim: DimensionScore) -> float:
    """超90天的数据降权10%"""
    if dim.stale:
        return dim.score * 0.9
    return dim.score
```

### 各维度数据新鲜度要求

| 维度 | 数据源 | 正常更新频率 | 过期阈值 | 降权幅度 |
|------|--------|------------|---------|---------|
| 质量分 | `--fund-perf` | 每日（净值更新） | 90天 | 10% |
| 成本分 | `--trade-rules` | 极少变动（修改才更新） | 365天 | 10% |
| 经理分 | `--fund-manager` | 季频（调仓期更新） | 90天 | 10% |
| 动量分 | `--fund-chart` | 每日 | 30天 | 10% |
| 聪明钱 | `holdings_snapshot` | 每日（管道运行） | 7天 | 20% |

## 五、反证条件（Falsify Rules）

评分引擎计算出的总分不能直接用于决策——必须经过反证规则过滤。以下任何一条触发，即使总分≥4.0，也必须**降低一级裁决**（buy→watch，watch→pass）或**直接否决**。

### P0：硬性否决

| 条件 | 说明 | 新的裁决 |
|------|------|---------|
| 规模 < 5000万 | 清盘风险 | pass |
| 经理换人（主动基金） | 历史评分对新人无效 | pass |
| 任何维度数据完全缺失（freshness > 365） | 该维度如同一堆常数 | 总分 cap 3.0 |
| 成立不足1年 | 无足够数据做质量/动量评估 | pass |

### P1：降级条件

| 条件 | 说明 | 新的裁决 |
|------|------|---------|
| Sharpe为负 **且** 动量分 < 2.0 | 基金处于持续下跌通道 | 总评分 ≥4.0 → watch；≥3.3 → pass |
| 质量分 < 2.0（主动基金） | 近1年+近3年排名均在后30% | 不走质量否决，但权重惩罚已体现 |
| 聪明钱分 = 0（有数据情况） | 大佬集体离场 | 总分 cap 3.3 |
| 规模过大（主动基金） | 规模过大影响超额收益 | 规模分非线性惩罚：100-200亿×0.9, 200-500亿×0.8, >500亿×0.7 |
| 3个以上维度 stale | 超过一半输入数据过期 | 总分降权 20% |

### 反证文档化

评分引擎输出时，`falsify_conditions` 字段记录所有被触发的反证规则（含检查时间），便于审计：

```python
fs.falsify_conditions = [
    "falsify: 规模<5000万 (当前3200万), 裁决: watch→pass",
    "downgrade: Sharpe=-1.2 + 动量分=1.5, 裁决: buy→watch",
    "stale: 3个维度过期 (质量90d/经理120d/聪明钱60d), 总分降权20%",
]
```

不做替换，只做**增强**。LLM 四大师评价**保留**，与评分引擎互补：

| 工具 | 职责 | 特点 |
|------|------|------|

| LLM 四大师评价（Claude Code） | **叙事解读** | "评分4.2但聪明钱分低是因为大佬在减仓，原因可能是..." |

评分引擎给数值，LLM给上下文。两者互补，不替代。

```

  fund-scan → 信号汇总 → 四大师LLM评价（文本） → HTML报告

增强后：
  fund-scan → 信号汇总 + 五维评分（数值） →
```

## 九、关键设计决策

### 为什么不做实时LLM评分？

每次让Claude Code用LLM生成评分成本高、不可复现、难以对比历史。五维评分是纯数值计算（Python函数），结果是确定性的：

- 同一只基金，同一时刻，评分相同
- 可以历史回溯——昨天评分4.2，今天变3.8，可以准确定位哪个维度下降了
- 可以作为 `fund-sell` 的量化输入（总分降≥0.3、动量降≥0.5）
- 可以在 HTML 报告中绘制评分变化趋势图

LLM的作用保留在**文本解读**——"评分4.2的基金，质量分5但动量分3，说明趋势在走弱"——这个才是 LLM 擅长的。

### 为什么权重这样分配？

（v1.1 调整后）

- **质量分25%最高**：过去业绩是最直接的 evidence
- **成本分20%+经理分20%**对称：低成本+好经理=长期持有基础。场外基金持有期以年计，管理费差异在复利下碾压短期信号
- **聪明钱20%**：大佬信号有用，但不应主导决策
- **动量分15%**最低：基金净值天然平滑，动量信号远弱于股票。基金不是日频交易品种

权重在第一个月试跑 + 评分稳定性校验后根据实际表现调整。
