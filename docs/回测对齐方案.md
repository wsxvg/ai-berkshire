# 回测与真实系统对齐方案

## 问题

现在回测引擎的买卖逻辑是自己手写的简化版，跟真实系统不一样：
- 回测卖点：动量<2.0 / 收益>30%止盈 / 亏损>15%止损
- 真实卖点：四级信号（🔴硬性卖出/🟡减仓/🟢止盈/🔵组合管理）+ 加权计算 + 买入护盾

## 方案：抽取共享模块，两边共用

```
tools/
├── fund_scorer.py        # 五维评分 ✅ 已共用
├── fund_planner.py       # 资金分配 ✅ 已共用
├── fund_sell_rules.py    # ← NEW: 卖出规则（从 SKILL.md 量化规则提取）
└── fund_buy_rules.py     # ← NEW: 买入信号逻辑

backtest/engine/
├── backtest.py           # 回测引擎 → import fund_sell_rules, fund_buy_rules
└── ...

skills/
├── fund-sell.md          # 引用 fund_sell_rules.py 而非自己描述规则
└── fund-scan.md          # 引用 fund_buy_rules.py
```

## fund_sell_rules.py 内容

从 fund-sell SKILL.md 提取可直接写代码的量化的规则：

```python

    """
    返回: {signal_level, signal_type, reduce_pct, reason}
    
    🔴 硬性卖出:
      - 大佬集体清仓≥3人 + 加权≥3 且 交易流水确认卖出
    
    🟡 减仓信号:
      - 单一大佬清仓 → 减1/3 (有买入护盾则跳过)
      - 多人减仓≥3 → 减1/3
      - 动量分<2.0 → 减1/3 (新增: 从fund_scorer取动量分)
    
    🟢 止盈:
      - 主动型: 收益>+50% → 止盈1/2
      - 指数增强: 收益>+30% → 止盈1/3
      - QDII主动: 收益>+50% → 止盈1/2
    
    🔵 组合管理:
      - 单只主动基金>15% → 减至10%
      - 调仓成本: 赎回费+申购费 > 预期超额×30% → 不调仓
    """
```

## 实施步骤

| 步骤 | 内容 | 时间 |
|------|------|------|
| 1 | 写 `fund_sell_rules.py` — 四级卖出信号函数 | 1h |
| 2 | 写 `fund_buy_rules.py` — 买入决策函数 | 0.5h |
| 3 | 回测引擎 import → 替换手写逻辑 | 0.5h |
| 4 | 跑一遍验证结果是否一致 | 0.5h |

## 好处

- 两边用同一份代码，不会出现"回测说能赚但实际不赚钱"
- SKILL.md 提示词可以直接说"调用 fund_sell_rules.check_sell_signals() 分析"而不是自己描述规则
- 后续改规则只改一个地方
