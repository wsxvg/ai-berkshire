### 4.5.2 分段验证（三段式）

```
段 A: 2023-07-01 ~ 2024-06-30（熊市+复苏早期，数据稀疏）
```

## 步骤 6：引擎拓展（逐个加，逐个验证，约 30 分钟）

**关键**：每个改动单独加、单独测。不允许一次加多个。

### 6.1 动态止损（P5.1）

持仓浮盈 >20% → 止损从 -30% 收紧到从高点回撤 15%。浮盈 >40% → 收紧到 10%。

改 Portfolio 类约 10 行。**只改这一个，不改其他**。跑一次冠军+动态止损。

### 6.2 波动率自适应仓位（P6）

`use_atr_sizing: True, atr_target_risk: 0.15`。

**不改止损逻辑。只加 ATR。** 跑一次。

### 6.3 跨资产对冲（P7）

QDII 敞口 >60% 且 CSI300 跌 >3% → 下次买入优先选非 QDII。

**不加 ATR、不加止损。只加对冲。** 跑一次。

### 6.4 基金质量过滤（P8）

经理变更 <6 月 / 规模 <1 亿 / 暂停申购 → 排除。

**不加前面三个。只加质量过滤。** 跑一次。

### 6.5 金字塔补仓（新方向）

引擎中 `pyramiding_enabled` 是僵尸参数——grid_search.py 里有但 backtest.py 从未实现。**需要手写逻辑。**

逻辑：持有某基金且浮亏 >5%，但大佬信号持续 → 不是止损，是越跌越买：

```
第一次买入：标准仓位 ×1.0
跌 5% 后仍有大佬买入 → 加仓 ×0.5
跌 10% 后仍有大佬买入 → 加仓 ×0.3
跌 15% → 不再加（防止深坑）
```

实现：在 Portofolio 的买入逻辑中加判断——已有持仓 + 浮亏 + 大佬共识 → 加仓。约 15 行。

**只加这一个改动。** 跑一次冠军+金字塔，看收益/回撤变化。

### 6.6 组合（仅组合通过前三门测试的改动）

如果 6.1-6.5 中有多个通过测试，组合在一起跑一次最终版。
