## 你的角色

你是一位有 10 年以上经验的基金投资专家 + 金融科技架构师，熟悉 AI Agent 辅助投资决策。你的任务是评审一份基金卖出体系的设计方案。

## 背景

### 项目定位

这是一个基于"大佬共识"筛选场外基金的投资研究框架。完整工作流：

```
fund-monitor → fund-checklist → fund-penetration → 买入决策
```

**第一期已完成**：从 0 到 1 构建了买入前的基金筛选和穿透分析体系。

**现在要做第二期**：从"买入决策"延伸到"买入→持有→卖出"的完整闭环。

### 已有基础设施

- **京东金融 API** (`jd_finance_api.py`)：已有 `--my-holdings`（自身持仓）和 `--holdings {uid}`（大佬持仓）两个接口
- **fund-monitor**：每日运行，能持续跟踪大佬持仓变化
- **fund-checklist**：买入前六关评分（已按基金类型分支：主动型走六关、纯被动走极简三关）
- **fund-penetration**：穿透到重仓股的三情景估值

### 原始项目（xbtlin/ai-berkshire）的参考

原项目有两个 SKILL 对应卖出场景：
- `thesis-tracker`：记录买入论文，定期检查是否被证伪
- `portfolio-review`：仓位审视与再平衡

我们的设计需要改编这两个概念到基金场景。

## 请重点评审以下问题

### 第一类：四级信号分类是否合理

1. **信号阈值是否合理？**
   - 硬性卖出：经理换人→清仓、规模<5000万→清仓、大佬集体≥3人清仓→清仓
   - 减仓观察：单一大佬清仓→减1/2、风格漂移→减1/3、跑输同类6个月→减1/3
   - 止盈：+30%止盈1/2、PE>历史90%分位止盈1/2
   
   这些阈值有没有过严或过松的？

2. **有没有遗漏的重要卖出信号？**
   - 比如：基金公司整体投研团队动荡、基金限购政策突变、费率突然上调
   - 还有什么常规基金卖出逻辑没覆盖到？

3. **"大佬集体卖出"信号是否可靠？**
   - 我们假设"大佬卖比买更认真"——这个假设在基金场景下成立吗？
   - 大佬可能因为自身资金需求（赎回买房、缴税）卖出，与基金质量无关，如何区分？

### 第二类：执行流程设计是否合理

4. **`fund-sell` 作为一个独立 SKILL 是否合适？**
   - 方案是"独立于 buy-side 流程，不修改现有 checklist/penetration"
   - 还是应该把卖出信号融入 `fund-monitor` 每日检查中更好？

5. **买入记录区的设计是否足够？**
   - 方案要求买入时记录：日期、净值、金额、目标收益、止损线、3句话理由、证伪条件
   - 3句话够不够？证伪条件清单应该谁来定？

6. **定期重估机制如何设计？**
   - 方案提出了"下次重估日期"概念（每季度或每6个月触发穿透重算）
   - 但也有"实时信号"（经理换人、大佬卖出）和"定期信号"两个时间维度
   - 这两种信号如何同时工作、谁的优先级高？

### 第三类：数据可行性

7. **京东金融 API 能否支持这些检测？**
   - `--my-holdings` 返回收益率，但能否返回"持有时间"用于止盈计算？
   - `--holdings {uid}` 能否支持"对比上期变化"（需要缓存机制）
   - 风格漂移检测需要的日频净值数据是否有接口？

8. **大佬持仓缓存方案**
   - 大佬集体卖出需要"上期 vs 本期"对比
   - 方案未详细说明缓存机制——每天全量拉一次，存本地 JSON 做 diff 够吗？

### 第四类：与现有体系的关系

9. **买入和卖出的"不对称性"是否合理？**
   - 买入需要六关/三关 + 穿透分析
   - 卖出只需要触发一个信号
   - 这种不对称在逻辑上成立吗？还是说卖出也应该走一个"A few checks"流程？

10. **止盈线和止损线谁来设定？**
    - 方案中默认目标收益+30%、止损-15%
    - 不同基金类型（指数基金 vs 主动基金）应该有不同阈值吗？
    - 不同买入策略（定投 vs 一次性 vs 分批）的止盈止损逻辑应该有差异吗？
