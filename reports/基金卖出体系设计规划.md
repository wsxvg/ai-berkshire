# 基金卖出体系设计规划

> **v2.0 — 已根据评审意见修订** | 评审评分：**B+**
> 
> 用于从"买入"扩展到买入→持有→卖出的完整闭环。
> 
> 前置文档：[基金场景下的AI-Berkshire适配分析.md](./基金场景下的AI-Berkshire适配分析.md)

---

## 一、为什么卖出体系是必要的

当前工作流的终点是"买入决策"：

```
monitor → checklist → penetration → 买入
```

但基金经理会换人、大佬在卖、规模会萎缩、穿透股会变贵——**买入那一刻的分析是有时效性的**。

关键事实：
- 京东金融 API 已有 `--my-holdings` 接口，可以返回自己的持仓（基金代码、收益率、持有金额、盈亏）
- 京东金融 API 已有 `--holdings {uid}` 接口，可以持续跟踪大佬的买卖动作
- fund-monitor 每日运行，天然适合承载卖出信号检测

## 二、设计哲学：借用原项目的两个 SKILL

| 原项目 Skill | 用途 | 我们的改造方向 |
|-------------|------|-------------|
| `thesis-tracker` | 记录买入论文，定期检查是否被证伪 | **基金版**：买入时记录"为什么买这只基金"（理由、目标收益、止损线），每日检查证伪条件 |
| `portfolio-review` | 仓位审视、再平衡、相关性 | **基金版**：检查单只仓位是否过重、跨基金持仓是否重叠、是否需再平衡 |

**两者合并为一个流程**：thesis-tracker 回答"要不要卖"，portfolio-review 回答"卖多少"。

### 补充设计原则

- **买卖不对称**：买入需要六关/三关 + 穿透分析，卖出一个信号就够。因为卖出的决策成本高于买入——卖出意味着认错或放弃未来收益。
- **全市场信号前置**：基金经理更换、规模预警、公司负面这三个信号不仅针对已持仓基金，也适用于候选基金，应该前置到 `fund-monitor` 每日运行中同步检测。
- **两类信号并行**：实时信号（经理换人、规模骤降、大佬卖出）为 P0 优先级，每日检查；定期信号（穿透重估、季度持仓变化）为 P1 优先级，按预设计划触发。实时信号检测到时就跳过定期信号。

## 三、卖出信号四级分类

### 🔴 第一级：硬性卖出（触发即执行，无需犹豫）

| 信号 | 适用基金类型 | 数据来源 | 行动 | 前置到 fund-monitor？ |
|------|------------|---------|------|:----:|
| **基金经理更换** | 主动型 / 指数增强 | `getFundManagerListPageInfo` → 对比历史经理 | **立即清仓**。主动基金/指数增强的核心是经理本人，换人等于换基金 | ✅ 是 |
| **基金规模<5000万** | 全部（不含纯被动指数） | `getFundDetailProfilePageInfo` | **立即清仓**。触及清盘高危线，强制赎回前必须跑。纯被动指数即使规模小也可正常交易，放宽 | ✅ 是 |
| **大佬集体清仓（≥3人清仓>80%）** | 全部 | `--holdings {uid}` → 检测"持有→0"或"持有→<20%" | **立即清仓**。多人同时清仓大概率有前瞻判断或内幕 | — |
| **基金公司重大负面（严重级）** | 全部 | 新闻/公告 | **立即清仓**。牌照吊销、重大违规处罚 | ✅ 是 |
| **基金被暂停申购/赎回** | 全部 | API/公告 | **立即清仓**。流动性冻结，能卖的时候立刻卖 | — |

> **不触发硬性卖出的情况**：纯被动指数型基金更换经理不触发（指数跟踪不依赖经理个人）；基金投资范围变更为减仓观察级别（变更流程极长，有时间反应）。

### 🟡 第二级：减仓观察（触发后按比例减仓）

| 信号 | 检测逻辑 | 行动 | 前置到 fund-monitor？ |
|------|---------|------|:----:|
| **单一大佬清仓** | `--holdings` 检测某个大佬从持有→0 | **减仓1/3**，同时查看该大佬是否也清仓了其他同类基金。单一大佬可能因资金需求（买房、缴税）卖出，权重不宜太高 | — |
| **大佬集体减仓（≥3人各减>30%）** | `--holdings` 检测多人同时大幅减仓 | **减仓1/3**，标记观察。如果有大佬B同期加仓，信号矛盾，不触发 | — |
| **风格漂移** | 净值与基准的滚动相关性偏离 | **减仓1/3**，重新评估基金定位 | — |
| **穿透股进入悲观情景** | 三情景估值重新计算后悲观概率>50% | **减仓1/2**，A股光模块类信号 | — |
| **持续跑输同类** | 连续6个月同类排名后25% | **减仓1/3**，减仓后若继续跑输3个月再减1/3 | — |
| **基金规模异常暴增** | 规模季度环比+200%以上 | **减仓1/3**，主动基金规模过大会稀释超额收益；被动基金规模暴增反而是好事（流动性提升），**不触发** | ✅ 是 |
| **基金投资范围变更** | 从"股票型"改为"混合型"，或新增衍生品 | **减仓1/3**，等于换了只基金，但变更流程极长（持有人大会+监管审批3-6个月），有时间反应 | ✅ 是 |
| **费率突然上调** | 管理费从1.5%调到2%等 | **减仓1/3**，直接侵蚀收益 | ✅ 是 |
| **基金限购政策突变** | 日限额从10万降到1000元 | **减仓1/3**，影响大额资金再平衡能力 | — |
| **基金公司投研团队动荡** | 核心研究员集体离职，经理"光杆司令" | **减仓1/3**，影响后续研究支撑 | ✅ 是 |
| **巨额赎回导致净值异常波动** | 单日赎回>10%规模 | **减仓1/3**，可能被迫卖出优质资产 | — |

### 🟢 第三级：止盈信号（触发后评估是否兑现）

止盈止损线**按基金类型和买入策略差异化设定**，不搞一刀切：

| 基金类型 | 目标止盈 | 止损线 | 止盈逻辑 |
|---------|:-------:|:-----:|---------|
| **主动股票型** | +50% | -20% | 主动基金超额收益空间大，放宽止盈；止损稍宽给经理操作空间 |
| **指数增强型** | +30% | -15% | 超额收益有限，到30%即可兑现部分 |
| **纯被动指数型** | +20% 或 按估值分位 | 不设止损 | 指数不会死，不设止损；止盈参考 PE/PB 历史分位 |
| **定投策略** | 基于持仓成本收益率 | 不设止损 | 定投分批买入，止盈按平均成本收益率算，不是按净值涨幅 |
| **一次性买入** | 基于绝对收益+估值分位 | 按类型上表 | 组合使用收益率目标和估值分位双重条件 |
| **分批买入** | 按各批独立止盈收益率 | 按类型上表 | 每批买入成本不同，独立计算止盈 |

**额外止盈信号**：

| 信号 | 检测逻辑 | 行动 |
|------|---------|------|
| **穿透股进入极度高估** | PE超过历史90%分位 | **止盈1/2**，等回调再买回（仅适用于穿透到单只核心股的情况；多只分散时单只PE高不代表整体贵） |
| **持仓时间+收益率双重确认** | 持仓>1年 且 年化收益率>目标年化 | **止盈1/3**，两个独立维度交叉确认才触发 |

### 🔵 第四级：组合管理信号（不单独触发卖出，但触发再平衡）

| 信号 | 检测逻辑 | 行动 |
|------|---------|------|
| **单只主动基金仓位>15%** | `--my-holdings` → 单只占比 | **减仓至10%** |
| **单只被动指数基金仓位>20%** | 同上 | **减仓至15%**，被动基金集中度风险低于主动 |
| **跨基金持仓重叠度>50%** | 多只基金前5重仓股Jaccard相似度 | **减仓重复度高的那只** |
| **更好的替代出现** | fund-monitor 新共识信号，评分高于当前持仓 | **评估是否调仓**，需计算调仓成本（赎回费+申购费+时间成本） |

> **调仓成本计算**：如果调仓的赎回费+申购费合计>预期超额收益的30%，不调仓。

## 四、执行流程设计

### 信号检测分工：fund-monitor 前置 + fund-sell 深度

评审意见采纳——**全市场级别信号前置到 `fund-monitor`**，持仓级别信号保留在 `fund-sell`：

| 信号归属 | 检测内容 | 输出类型 |
|---------|---------|---------|
| **`fund-monitor` 前置检测**（每日运行，全市场扫描） | 经理更换、规模<5000万（含所有候选基金）、规模暴增、费率上调、投研动荡、基金公司负面、基金范围变更 | 🔴 红色预警 / 🟡 黄色预警 → 写入 shared 预警池 |
| **`fund-sell` 持仓检测**（每日运行，只检查已持仓） | 大佬买卖变化、风格漂移、穿透股重估、跑输同类、止盈、组合管理、限购突变、巨额赎回 | 卖出/减仓/持有 具体建议 |

**联动逻辑**：fund-monitor 检测到红色预警时，自动触发 fund-sell 对该基金执行全部信号检查。

### 新增 SKILL：`fund-sell.md`（归入持仓管理类）

**定位**：买入+持有后，每日检查是否需要卖出。

**输入**：
- `--my-holdings` → 当前持仓列表（代码、收益率、金额、首次买入日期）
- `--holdings {各uid}` → 大佬当前持仓变化（每日缓存对比）
- `fund-checklist` 缓存 → 上一次的评分 + 买入记录
- `fund-penetration` 缓存 → 上一次的穿透估值 + 下次重估日期
- `fund-monitor` 预警池 → 前置检测到的共享预警信号

**执行流程**：

```
每日触发
     ↓
1. 读取 fund-monitor 预警池（前置信号已检测好）
     ↓ 检查共享预警池中是否有持仓基金
2. 如果有红色预警 → 立即清仓，输出报告
   ↓
3. 读取自身持仓（--my-holdings）
   ↓
4. 读取大佬持仓缓存，增量对比变化
   ↓
5. 对每只持仓基金，顺序检查四级信号：
   - 已触发红色预警的基金跳过，直接清仓
   - 🟡 减仓信号 → 按比例减仓，输出报告
   - 🟢 止盈信号 → 按类型差异化止盈，输出报告
   - 🔵 组合管理信号 → 再平衡建议，输出报告
   ↓
6. 更新大佬持仓缓存（存入当天快照）
   ↓
7. 输出合并报告：持有/减仓X%/清仓 + 理由 + 建议金额
```

## 五、大佬集体卖出的特殊处理

这是最实用的信号之一。因为：

**大佬买可能是跟风，但大佬卖一定是认真考虑过的。** 卖出的决策成本高于买入——卖出意味着认错（亏钱卖）或放弃未来收益（赚钱卖），两个都不轻松。

**但"大佬卖比买更认真"这个假设有局限**：大佬可能因资金需求（买房、缴税、家庭支出）赎回，与基金质量无关；可能有固定季度/年度再平衡习惯；可能因风格轮动（成长→价值）卖出成长型基金，不代表成长型基金变差。

### 检测逻辑

```
每日检查 focus_uid_list 中每个人的持仓变化：
  对比上期缓存 vs 本期，找出所有"持有→0"或"持有→<20%"的基金
  对每位大佬按持有时间加权：
    持有>2年的大佬 → 权重 × 2
    持有6个月~2年 → 权重 × 1（基准）
    持有<6个月 → 权重 × 0.5（可能是短期交易，非长期判断）
  
  加权后分档预警：
    加权人数≥3人清仓 → 红色预警 → 自动触发减仓
    加权人数≥2人清仓 → 黄色预警 → 标记观察
    加权人数≥2人减仓>50% → 黄色预警 → 标记观察
```

### 误报排除（完善版）

- **周期性调仓排除**：排除固定季度末调仓模式（如每年3月/6月/9月末的规律性调整）
- **资金需求排除**：如果大佬A清仓单只基金但其他持仓不变，可能是资金需求而非质量判断；如果清仓多只基金（3+），更可能是策略转向，信号权重提高
- **买入抵消**：大佬A清仓但大佬B同期加仓 → 信号矛盾，不触发卖出，标记"分歧观察"等待后续信号
- **区分清仓 vs 减仓**：清仓>80%或持有→0才是强信号，减仓<20%可能是日常再平衡，不触发预警
- **区分"长期持有者卖出"和"短期交易者卖出"**：上述持有时间加权机制自动处理此逻辑
- **全市场同步清仓**：如果当天基金净值大幅下跌+多人清仓 → 可能是恐慌性赎回，不是主动卖出判断，需要结合净值走势分析

## 六、与原项目 portfolio-review / thesis-tracker 的关系

| 原项目 | 我们的实现 | 差异说明 |
|--------|----------|---------|
| `thesis-tracker` 写长篇买入论文 | **简化为3句话买入理由 + 系统自动生成证伪条件清单** | 基金不需要万字论文；证伪条件由 checklist 评分时自动提取关键风险点（如"经理任职<3年→经理更换即证伪"），用户可修改 |
| `portfolio-review` 全组合再平衡 | **分级处理：仓位过重→减仓 / 持仓重叠→减仓 / 替代出现→调仓** | 目前持仓数量少，暂不做全组合优化，只做减法 |
| `portfolio-review` 相关性分析 | **跨基金持仓重叠度分析（Jaccard相似度）** | 基金特有的需求——多只基金可能在同一只股票上押注过多 |

**关键原则**：
- 卖出体系是**独立的 skill**（`fund-sell.md`），不修改现有 buy-side 流程。
- 买卖不对称：买入需要六关/三关，卖出只需要触发一个信号就够。
- **全市场级别信号前置**到 `fund-monitor.md`，不混入 sell 逻辑。
- 止盈止损阈值按基金类型差异化设计，不自作聪明一刀切。
