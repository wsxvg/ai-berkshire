# 基金场景下的 AI-Berkshire 适配分析

> 说明：此文档梳理了从原始项目（xbtlin/ai-berkshire）到场外基金监控/筛选场景的改动逻辑，供 AI 评审分析。
> 原始项目：一个基于 Claude Code / Codex 的个股价值投资研究框架，含 18 个 Skill + 金融工具集。
> 我们的场景：监控和筛选场外基金，通过"大佬共识"发现信号，穿透到持仓股验证。

---

## 一、场景差异对框架的影响

### 核心差异矩阵

| 维度 | 原始项目设计（个股） | 我们的场景（基金） |
|------|-------------------|------------------|
| 分析对象 | 一家上市公司 | 一只基金 → 穿透到持仓股 |
| 研究深度 | 7模块×4大师，万字报告 | 先六关快速筛选，好的才穿透 |
| 数据源 | 需要2个独立来源交叉验证 | 京东金融唯一源即可（基金数据中心化发布） |
| 决策输出 | 买入/不买/灰色，带价格区间 | 定投/分批/大额/等回调，带金额和频率 |
| 核心风险 | 公司基本面恶化、管理层诚信、估值陷阱 | 基金经理更换、规模清盘、限额不足、底层太贵 |
| 研究频率 | 一次性深度研究 + 定期跟踪 | 每日监控 + 每周/每月筛选 |

### 为什么原始项目的"双源验证"在基金场景不适用

股票数据涉及多交易所、多数据商、多币种，存在单位搞错、数据不一致的风险。
基金的核心数据（净值、规模、持仓、基金经理）全部由基金公司统一发布，各平台的差异仅在于更新延迟，不存在"两个来源对不上"的问题。
因此京东金融作为唯一数据源完全够用——它覆盖的基金种类比支付宝还全。

## 二、原始项目 18 个 Skill 的适配评估

### 🔴 完全不在场景内（6 个）

| Skill | 原用途 | 不适配原因 |
|-------|--------|-----------|
| `private-company-research` | 研究SpaceX等未上市公司 | 基金不投未上市公司 |
| `deep-company-series` | 8篇长文拆一家公司，公众号发布 | 我们不产出公众号内容 |
| `management-deep-dive` | 分析CEO的诚信度、资本配置 | 基金经理不适合用这套标准（需要的是稳定性+风格一致性，不是"老板值不值得信任"） |
| `bottleneck-hunter` | 找供应链瓶颈（如铀矿、光刻机） | 行业筛选场景，和基金筛选无关 |
| `earnings-team` | 四大师并行读财报+编辑润色+发布 | 多个Agent发文，我们用不上 |
| `wechat-article` | 微信公众号文章 | 不需要 |

### 🟡 偶尔有用但优先级低（5 个）

| Skill | 可用场景 | 备注 |
|-------|---------|------|
| `industry-research` | 研究"AI算力基金"等行业层面的机会 | 目前已有"大佬共识"作为信号源，暂时不需要从行业层开始筛 |
| `industry-funnel` | 全市场→粗筛→精选→3家 | 同上 |
| `news-pulse` | 某只基金/重仓股暴跌时快速归因 | 场景存在但频率低 |
| `dyp-ask` | 段永平式思维工具 | 辅助参考，非流程必须 |
| `earnings-review` | 精读个股原始财报 | 只在穿透到核心重仓股时需要，且只读关键季报即可 |

### 🟢 有用但需要改造（4 个）

| Skill | 原版用途 | 改造方案 |
|-------|---------|---------|
| `quality-screen` | 7条硬指标排除垃圾股 | **改造为基金版**：规模<2亿排除 / 经理年限<2年排除 / 成立<1年排除 / 费率>1.5%减分——实际上已经融入 fund-checklist 的各关标准中 |
| `investment-research` | 7模块四大师深度分析 | 穿透到英伟达这类核心股时，应该用它做深度分析（比 investment-checklist 更细），但目前只用了 checklist |
| `portfolio-review` | 仓位审视、再平衡、相关性 | 买入后阶段才用得上，目前还没到这一步 |
| `thesis-tracker` | 记录买入论文，定期检查是否被证伪 | 买入后阶段用，目前还没到 |

### 🔵 必须用的（2 个）

| Skill | 使用情况 |
|-------|---------|
| `investment-checklist` | ✅ 穿透分析时的六关框架（能力圈→好生意→护城河→管理层→安全边际→纪律→镜子测试） |
| `financial-data` / `financial_rigor.py` | ✅ 精确十进制计算、三情景估值（乐观/中性/悲观） |

## 三、我们新增的 4 个基金专用 Skill

### 1. `fund-monitor.md` — 大佬持仓监控（入口层）

**定位**：每日/每周运行的入口 skill。
**输入**：京东金融 API 拉取 11 位基金博主的最新持仓。
**输出**：共识信号排行（按持有者数量 + 合计金额排序）。

**核心新增逻辑**：
- 共识信号强度分级（⭐⭐⭐≥8人 / ⭐⭐5-7人 / ⭐4人）
- 各大佬持仓概况
- 按持有者数量排名的候选基金列表

### 2. `fund-checklist.md` — 基金壳六关评分（分析层）

**定位**：对候选基金做"基金本身"的质量检查。
**六关**：
| 关卡 | 评分项 | 参考标准 |
|------|--------|---------|
| 能力圈 | 策略是否清晰、投向是否可控 | 全球科技/A股光通信等易理解 |
| 历史业绩 | 近1年/3年/5年收益排名 | 同类前30%加分 |
| 基金经理 | 任职年限、换手次数 | ≥3年加分，频繁换人减分 |
| 费率 | 管理费+托管费+申赎费 | 越低越好 |
| 流动性 | 日限额、T+N到账 | 大额场景关注日限额 |
| 聪明钱验证 | 多少大佬持有 | ≥5人加分 |

**输出**：综合评分（5分制）+ 具体建议（定投/分批/大额/等回调/不买）。

**为什么需要这个**：原项目的 `quality-screen` 只有简单的"7条排除"，不够细。基金的"壳质量"（经理/费率/限额）和"底层资产质量"是两层问题，需要分开评估。

### 3. `fund-penetration.md` — 基金穿透分析（穿透层）

**定位**：把 Checklist 评分≥3.5 的基金，穿透到前 5 大重仓股做估值验证。
**核心发现机制**：
- 提取高分基金的前5持仓
- 按集中度排序，选出出现频率最高的股票
- 对核心股执行 investment-checklist 六关分析
- 如果穿透股太贵 → 修正基金建议（即使基金壳好也等回调）

**实际案例（2026-06-28）**：
- 011452 基金壳评分 4.10（高分），但穿透发现重仓新易盛（PE 76x）和中际旭创（PE 83x）
- 三情景估值显示中性情景下分别要亏 34% 和 49%
- 结论：基金很好，但现在买底层太贵了，等回调

### 4. `fund-quarterly.md` — 季度持仓更新检查

**定位**：每季度基金公布最新持仓时，对比上季度变化。
**尚未实际使用**。

### 原始项目对比：新增 skill 的功能分布

```
原始项目（个股）             我们的改造（基金）
─────────────────           ─────────────────
monitor（无）               → fund-monitor（新增）
quality-screen（7条排除）   → fund-checklist（六关评分）
investment-checklist       → fund-penetration（穿透层复用）
（无）                     → fund-quarterly（新增）
```

## 四、工作流对比

### 原始项目工作流

```
/industry-funnel → 全市场筛选
      ↓
/investment-checklist → 六关快速过
      ↓
/investment-research → 7模块深度分析
      ↓
/portfolio-review → 仓位管理
      ↓
/thesis-tracker → 持有期跟踪
```

设计哲学：**先筛行业，再筛个股，深度研究，长期持有。** 每一步都是一个人面对全市场。

### 我们改造后的工作流

```
fund-monitor → 大佬共识发现（11人的持仓帮你筛市场）
      ↓
fund-checklist → 基金壳六关评分（只筛基金质量）
      ↓
fund-penetration → 穿透到核心股，investment-checklist验证（检查底层贵不贵）
      ↓
决策：定投/分批/大额/等回调
```

设计哲学：**借力。** 不是一个人面对全市场，而是看11个"聪明钱"的重叠区在哪，然后只分析重叠区。

### 核心差异总结

| 环节 | 原始项目 | 我们的改造 |
|------|---------|-----------|
| 从哪里开始筛 | 全市场（数千只股/基金） | 大佬持仓（50-80只基金） |
| 第一层筛什么 | 行业/赛道 | 共识信号强度（几个人持有） |
| 第二层筛什么 | 公司质量 | 基金壳质量 |
| 第三层做什么 | 深度个股研究 | 穿透到持仓股验证估值 |
| 决策什么 | 买不买这只股票 | 买不买这只基金 |
| 持有期管理 | 论文跟踪+组合再平衡 | 季度持仓更新检查 |

## 五、关键设计决策记录

### 决策 1：为什么不逐只基金做深度研究？
原始项目的 investment-research 对一只股票输出万字报告。20只基金×每只前5持仓 = 100只股票，不可能每只都深度研究。
用 fund-checklist 六关（每只10分钟）快速过 → 只有评分≥3.5的才穿透 → 穿透只做共识度最高的3-5只持仓股。
**效率优先于深度。**

### 决策 2：为什么需要两层"壳 vs 底"？
两个维度可以是正交的：
| | 壳好（经理稳/费低/不限购） | 壳差 |
|--|--------------------------|------|
| **底层便宜** | ✅ 最佳买入时机 | ❌ 不买 |
| **底层贵** | ⚠️ 等回调再买 | ❌ 不买 |
011452 基金评分 4.10（壳好），但底层光模块太贵 → 结论是"等回调"而不是"不买"。如果没有穿透层，就会错过这种信号。

### 决策 3：为什么不沿用"两个数据源"规则？
股票有港股通/美股/A股多交易所，币种单位容易出错（港币亿 vs 人民币亿）。
基金净值、规模、持仓、经理都是基金公司统一发布的，各平台差异仅在于更新延迟（京东金融通常比天天基金快1天），不存在"数据打架"的问题。京东金融覆盖基金数比支付宝还全，单一来源完全够用。

### 决策 4：为什么持股穿透用 investment-checklist 而不是 investment-research？
两者都是原项目的 skill。但：
- checklist 是六关过筛，适合快速判断"值不值得继续看"，10-15分钟完成
- research 是7模块深度分析，需要1-2小时
穿透分析的目的是**验证估值是否合理**，不是写一篇完整的投资研报。Checklist 的五关（能力圈→好生意→护城河→管理层→安全边际）已经覆盖了核心判断维度。

## 六、当前局限与改进空间

### 已知局限

1. **持股数据滞后**：基金季报有1-2个月延迟，此时买入可能基金已经调仓
2. **数据源单一**：虽然京东够用，但如有异常值无法交叉校验
3. **无持仓变动检测**：大佬可能已经调仓但我们不知道（除非重新拉一次API）
4. **无组合管理**：买了多只基金后，没有系统化的相关性分析和再平衡建议

### 建议改进优先级

| 优先级 | 改进项 | 说明 |
|:------:|--------|------|
| P0 | 穿透深度加强 | 当穿透股是英伟达这种级别的核心资产时，调用 investment-research 做7模块深度分析，而不是只用 checklist |
| P1 | 买入后管理 | 实现 portfolio-review（仓位集中度） + thesis-tracker（论文跟踪）的基金版 |
| P2 | 季度持仓变化检测 | 补上 fund-quarterly 的实际使用 |
| P3 | 异常检测 | 如果某只基金规模突然暴增或经理换人，自动标记重新评估 |
