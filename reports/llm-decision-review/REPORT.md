## 任务 1：补历史新闻快照

### 计划
用 cookie 拉 JD API，按日期归档到 `data/fund_cache/daily_news/{YYYY-MM-DD}.json`

### 实际结果：⚠️ **有限度完成**

**问题 1（API 限制）**：JD 新闻 API 不返回绝对日期，只返回 "X小时前" 这种相对时间。
**问题 2（无历史 API）**：JD 不提供"截至 X 日的新闻"接口，只能拉当下看到的。

**解决**：把 7-11 真实快照**复制**为 6-01 ~ 7-10 的代理快照，每个标 `is_asof_proxy=true` + `_warning`。

**重要警告**：LLM 决策时必须看到这个标记，知道这是代理数据，不是真实的 6-01 新闻。

**未完成项**：
- 没有为每个日期真正拉过新闻（JD 不支持）
- 代理快照里包含 7-11 拉到的"6 小时前"新闻 = 实际是 7-11 凌晨发的——对 6-01 来说是"未来"信息
- LLM 决策若用了这些新闻，**仍存在未来函数**（虽然不是直接的，但 7-11 视角的新闻描述 7-11 的事）

## 任务 2：LLM 决策回测

### 计划
跑 6-01 ~ 7-11 每天，用 4 大师基金版决策，对比 actual_buys

### 实际结果：❌ **命中率 20% (2/10)，但暴露了多个严重问题**

#### 问题 1: 评分模型缺陷

**第一版** LLM 4 大师评分全部 0 分（4 个维度独立加/减，无基础分），完全没意义。
**第二版** 加基础分 3.0，但**没考虑业绩排名**——结果 LLM 选 013841/024663/024239，actual 选 539002/018147/501226/016664/021842/017731/024239/024663。**2/8 重合 = 25%**。
**第三版** 加 ret_60d 到巴菲特视角，**6-01 命中 2/5 = 40%**（024239 024663 重合）。

**根本原因**：4 大师评分模型是**简化版**，`score_fund_backtest` 用了 1 年/3 年排名、夏普、回撤、估值等多维度，我没法完整模拟。

#### 问题 2: 组合约束时序问题

**6-01** actual 跑时 `holdings={}`（pending 中，6-02 settle），LLM 看到 `holdings=0` → 可买 6 只
**6-02** actual 跑时 `holdings=1`（501226 6-02 settle），但 LLM 6-02 也用 `holdings=1` 算相关性
**6-03** actual 跑时 `holdings=6`（5 只 settle）→ 相关性 > 0.85 过滤掉几乎所有 → 没 buys

**问题**：6-02 时 daily_live 实际买了 5 只，但 LLM 算出 TOP3=[]（因为相关性过滤太严）。**实际 daily_live 也是同样问题**，但它的相关性计算用 portfolio.holdings 状态有差异。

**核对**：6-02 daily_live 真买了 5 只：024239/016664/013841/017731/501226。**意味着 daily_live 6-02 跑时相关性和 LLM 计算的不一样**——LLM 严格按 ground truth holdings 算，但 daily_live 用 portfolio.holdings（可能含 pending_buys）。

#### 问题 3: 大佬信号数据稀疏

`trading_history_fixed.json` 实际只有 2026-07-01 之后的 1 个月数据（1094 条），2024-03 ~ 2026-06 是 7762 条但主要是模拟。

**6-01 LLM 看 smart_net**：13 只基金中**所有都是 0**（因为 6-01 距 2024-03 已 27 个月，30 天窗口内没有大佬交易）。

这跟 `fund-monitor` 真实跑出来的 "大佬近 30 天净买卖 0" 一致，**说明 LLM 框架和 daily_live 都面临这个数据稀疏问题**。

#### 问题 4: 反未来函数强制

我 LLM 决策时**严格**做了：
- `m = calc_metrics(code, asof)` 用 `<=asof` 过滤 chart
- 大佬交易用 `<=asof` 过滤
- 基金档案用最新（**可能轻微未来**，因为不知道 6-01 时该基金的规模）
- 新闻用 6-01 代理快照（**实际是 7-11 数据**）

**基金档案反未来函数不严格**。`fund_profile_013841.json` 是当下抓的，可能含 6-01 之后的信息（经理换人、规模变化）。

## 任务 3：诚实结论

### 收益对比

| 策略 | 6-01 → 7-01 收益 | 说明 |
|------|------------------|------|
| 机器 baseline (`daily_live.py`) | **+10.35%** | 23 天真实模拟, 6 只持仓 |
| LLM 4 大师（我脚本） | **N/A** | 只买了 2 只 (6-01 选出的), 但实际只算 1 只成功买入 |
| 假设 LLM 也跑 daily_live 同样约束 | 待 LLM 真实运行测试 | 见下方 |

### 关键发现

1. **JD 不提供历史新闻 API**——按日期归档只能代理，**存在未来信息**风险
2. **4 大师评分简版**无法匹敌 `score_fund_backtest`（后者用了 1/3/5 年排名、夏普、回撤等多维度）
3. **组合约束时序**复杂：pending_buys / settle / 相关性计算都有时间差
4. **基金档案反未来函数不严**：`fund_profile_*.json` 是当下抓的
5. **LLM 决策命中率 20%**：比纯随机好，但不优于机器 baseline
6. **机器 baseline +10.35% 的本质**：6 月 QDII/科技主线 + 评分系统偏好动量 + 仓位 34%——**不是"决策准"，是"选对了板块"**
