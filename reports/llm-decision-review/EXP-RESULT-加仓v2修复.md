## 3. 加仓为什么"加了但没用"

### 3.1 修复前 (没加仓触发)
- LLM 5-27 才买入 013841 (机器 4-28 已买)
- 错过 30 天涨幅
- 总成本 33,300 → 7-11 净值 119,272 (+19.27%)

### 3.2 修复后 (加仓 7 次)
- 5-27 买 013841 (晚 30 天) + 5-28 加仓 2,200
- 5-29 买 024663 (晚 3 天) + 6-03 加仓 4,500
- 5-25 加仓 022184
- 总成本 42,500 → 7-11 净值 112,431 (+12.43%)

**核心问题**:
- 加仓金额小 (单笔 300-4,500) 累计只加了 9,200
- 但**晚建仓的损失 = 9,200 远大于** (机器建仓时机更早)
- 加仓补不回"错过建仓"的损失

### 3.3 机器多赚的钱

机器 6-22 买了 501226 (500 元), 7-11 浮盈 +0.62% = 3 元. LLM 否决了.

但机器**真正的优势**是 4-28 早买 013841 (17,800 → 35,193 浮盈 +97.70%):
- 4-28 净值: yAxis=64.7 (累计 +64.7%)
- 7-11 净值: yAxis=127.7 (累计 +127.7%, 即 +63% 区间涨幅)
- 17,800 × 1.63 = 29,014 (相对建仓 17,800 的 11,214 利润)

LLM 5-27 买 013841 时 yAxis=112.4 (累计 +112.4%), 到 7-11 = +127.7% 区间只涨 15.3%:
- 17,800 × 1.15 = 20,470 (相对建仓 17,800 的 2,670 利润, 加加仓 2,200×1.15=2,530 → 总 5,200 利润)
- 净少赚 11,214 - 5,200 = 6,014 → 6.84% 总资产差异

## 4. 关键发现

### 🚨 发现 1: 加仓触发没问题, 但 LLM 错杀 013841 损失远大于加仓收益

**修复加仓触发 = 解决了一个本就不存在的问题** (机器 baseline 60 天 0 次加仓, 因为没有错杀, 没必要加仓).

LLM 的核心问题不是"不会加仓", 而是"错杀冠军延迟建仓". 加仓补不回延迟成本.

### 🚨 发现 2: 加仓信号 (5d_dd/5d_chg) 有效

3 个月触发了 7 次加仓 (024239 3 次, 022184 2 次, 013841 1 次, 024663 1 次). 频率合理 (< 1 次/月/基金).

但加仓金额太小 (300-4,500), 总体贡献有限.

### 🚨 发现 3: 30 天频次限制生效

加仓 024239 间隔: 04-17 → 05-18 → 06-17 = 31 / 30 天 (刚好过线)
加仓 022184 间隔: 05-25 → 06-24 = 30 天 (刚好过线)

频次限制正确执行, 没出现"天天加仓"的复利 bug.

## 5. 改进方向

### 方向 1: 改 LLM prompt 不再错杀 (最高优先级)

v3 prompt 应该删除"同主题"否决:
- ✅ 规模 < 5 亿 (501226 案) — 保留
- ✅ 经理刚变更 — 保留 (新加)
- ✅ 单笔 > 25% 仓位 — 保留
- ❌ ~~同主题重复~~ — 砍冠军
- ❌ ~~持仓集中度~~ — 机器相关性已管

**预期效果**: LLM 4-28 买 013841 → 加仓触发 → 总成本类似机器 → 收益接近 +19.27%

### 方向 2: 加仓金额放大 (保守做)

当前加仓金额 = 现市值 × 0.3, 太保守. 可改为:
- 信号 1 (突破): 现市值 × 0.5
- 信号 2 (抄底): 现市值 × 0.5 (但需 60 日趋势 > +5%)

但要小心 30 天频次限制 → 1 年最多 12 次加仓, 单只最多 4-5 次.

### 方向 3: 不接 LLM (干净解)

如果 LLM 错杀 1 次冠军 = -6% 总收益, **不如不接 LLM**. 机器 baseline +19.27% 已足够好.

LLM 真正价值:
- 看公告 (经理离任) — 这是机器看不到的
- 报告可读性

但**否决权 = 砍冠军风险** — 不是一个好设计.

## 7. 答复你的两个问题

**Q1: 加仓改造后收益提高了还是降低了?**
- 比没加仓的 LLM (+10.22%) **提高 +2.21%** (到 +12.43%)

- 净影响: 修复加仓触发让 LLM 不再"白白错过机会", 但补不回 LLM 错杀的损失
