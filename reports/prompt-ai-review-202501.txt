### 3. fund-sell — 双缓存合并加权

**数据流**：
```
holdings_diff_cache.json (已有)  ─┐
                                  ├─→ 加权计算 → 卖出建议
trading_records_cache.json (新增) ─┘
```

**加权公式**：
```
weighted_clear = holdings_score × (1 - sell_count × 0.1)
sell_count=0 → 满权重
sell_count=1 → ×0.9
sell_count=2 → ×0.8
sell_count≥3 → ×0.7（上限）
```

**硬性卖出（🔴）独立触发条件**：
```
if net_signal == "sell" AND sell_count >= 2 AND 持仓金额减少:
    → 强制执行，不依赖加权分
```

**Buy Shield（🛡️）**：
```
if net_signal == "buy" AND buy_count >= 2:
    → 持有确认，跳过本轮减仓检查
```

**信号矛盾处理**：
```
如果 holdings_diff 显示减仓，但 trading_records 多人买入
    → 标记"分歧观察"，不触发减仓，等待后续信号清晰
```

**评审重点**：
- sell_count × 0.1 系数的依据？是否需要可配置化？
- sell_count ≥ 2 独立触发硬性卖出是否过于敏感？
- buy shield 无时效限制是否合理？（买入信号是否应该随时间衰减？）
- "分歧观察"的处理是否足够保守？
