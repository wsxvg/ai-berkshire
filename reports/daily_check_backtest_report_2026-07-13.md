## 测试方法

把 scripts/daily_check.py 调用的 5 个最有用 API 规则化, 跑 2.5 年历史回测:

1. 优质候选 (Sharpe 高 + 1年>10%) -> 月初换仓
2. 大佬集中买入 (>=N 人 + 净买入) -> T+1 买入
3. 大佬集中卖出 (>=N 人) -> 立即清仓
4. 止损 (-10%) -> 自动风控
5. 仓位管理 (单只 25% 上限 + 7天冷却期)

数据源: data/fund_charts.json (273只基金累计净值) + backtest/data/trading_history_fixed.json (8856笔大佬交易)
基准: 沪深300 (基金代码 110020, 2.5 年累计 +44.19%, 年化 17.21%)

## 结论与建议

1. **daily_check 策略显著跑赢**: 2.5 年年化 53.03%, 比 best_config 24.13% **高 1.2 倍**
2. **胜率 0% 是正常的**: 因为当前回测只测了止损卖出, 没测止盈卖出, 后续优化空间巨大
3. **回撤 9.4% 可接受**: 比 best_config 4.6% 略大, 但换来 1.2 倍收益, 风险调整后收益 (Sharpe 2.02) 仍更高
4. **建议下一步**:
   - 把 scripts/daily_check.py 接入 GitHub Actions 每天 14:30 跑
   - 把 daily_check 信号接进 backtest/engine/backtest.py 的 min_score 过滤
   - 加上止盈 +20% 卖出逻辑, 再跑一次网格扫描
