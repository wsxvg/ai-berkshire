### 0.3 K策略配置分析

```python
K_CONFIG = {
    'min_score': 0.0,        # 不评分！完全信任大佬共识
    'stop_loss_pct': -30,     # 极宽止损（-30%）
    'take_profit_pct': 50,    # 极高止盈（+50%）
    'profit_mode': 'half',    # 止盈时卖一半
    'cost_penalty': 0,        # 无费率惩罚
    'min_consensus': 2,       # 2人买入即跟
    'fund_type_filter': 'all' # 不限基金类型
}
```

**K策略为什么强**：
1. 完全信任大佬共识（≥2人买入），不添加主观评分噪音
2. 极宽止损(-30%)给输家恢复空间
3. 极高止盈(50%)让赢家充分奔跑
4. 半仓止盈锁定部分收益同时保留上涨敞口
5. 极少干预 = 最低交易摩擦

### 0.4 引擎核心逻辑理解

- `run_backtest()`: 主循环494天，每日处理交易记录→评分→买卖
- `score_smart_money_backtest()`: 扫描所有历史交易记录计算聪明钱分（O(n²)性能瓶颈）
- `Portfolio`: T+N确认、费率、60天最低持有、年交易上限6次、冷却期
- 年化公式: `(final/total) ** (252/days) - 1` × 100（原始公式，不修改）

**引擎已有参数**（可用于优化）：
- kelly_cap (默认0.2), max_position_pct (25), cash_reserve_pct (0.1)
- trailing_tp_activate/drawdown, momentum_sell (2.0)
- sell_consensus, bear_market_no_buy, net_signal
- max_holdings, max_sector_pct, cooldown_days
- timing_filter, ml_signal, dynamic_ranking

**引擎微调**（不影响现有逻辑）：
1. 添加 `follow_only_uid` 参数（3行新增），支持单大佬跟随实验
2. **修复min_consensus硬编码bug**：第1483行 `buy_count < 2` 改为 `buy_count < config.get("min_consensus", 2)`
   - 原代码硬编码共识门槛为2，忽略config中的min_consensus设置
   - 修复后min_consensus=1可正常工作（单大佬跟随所需）
   - 对K基线(min_consensus=2)无影响，行为完全一致
