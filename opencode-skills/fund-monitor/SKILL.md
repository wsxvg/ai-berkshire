# 场外基金大佬持仓监控

监控 11 位关注大佬的**持仓变化 + 交易流水**，识别共识信号，自动触发基金分析。

**核心原则**：交易流水（大佬正在买入/卖出什么）权重大于持仓快照（大佬持有什么）。
因为持仓可能买了很久（成本未知），而交易流水反映**当下的判断**。

**支持参数**：`--user {uid}` 仅监控指定用户 | `--offline` 离线模式 | 默认全量监控

### 第四步：获取交易流水（信号核心）

> 持仓快照是滞后数据（不知道自己买于何时），**交易流水**才是大佬真实操作的记录。

对每个关注人（或全量），拉取近期交易流水：

```bash
python3 tools/jd_finance_api.py --trading-records jimu_user_info-{uid}
```

#### 交易流水解析

每条交易记录包含：

| 字段 | 含义 | 示例 |
|------|------|------|
| `action` | 操作类型 | 买入 / 卖出 / 调仓 |
| `fund_name` | 基金名称 | 中欧医疗健康混合C |
| `amount` | 金额 | +5000元 / -3000元 |
| `detail` | 操作说明 | 定投买入 / 手动加仓 |
| `summary` | 用户动态描述 | 今天加仓了这只基 |

#### 交易流水 → 信号转换

| 原始记录 | 转化为信号 | 信号强度 |
|---------|-----------|---------|
| 大佬A 买入 基金X {金额} | **大佬A最近买了X** — 真实买入信号，参考价值最高 | ⭐⭐⭐ |
| 大佬A 卖出 基金X {金额} | **大佬A在减仓X** — 止盈或止损，需排查原因 | ⚠️ |
| 多人（≥3）同期买入同一基金 | **强共识买入** — 多个筛选体系同时选中，极大概率是好基金 | ⭐⭐⭐ 最强 |
| 同一人反复买入同一基金 | **持续加仓** — 越跌越买，代表极高置信度 | ⭐⭐ |
| 多人（≥2）同期卖出同一基金 | **分歧/撤退信号** — 可能已到天花板 | 🔴 红色 |

> **关键原则**：交易流水的信号权重大于持仓快照。
> - 持仓里持有=可能买了很久了（成本可能很高或很低），不代表现在的判断
> - 交易记录里的买入=**现在正在买**，代表当下的判断
> - 交易记录里的卖出=**现在正在卖**，比持仓里"不见了"更及时（清仓可能发生在两周前）

#### 交易流水数据持久化

交易流水分析结果写入 `data/trading_records_cache.json`，供 `fund-sell` 读取：

```json
{
  "date": "2026-06-28",
  "funds": {
    "006105": {
      "code": "006105",
      "name": "XXX基金",
      "buy_actions": [
        {"name": "蓝鲸跃财", "amount": "+5000", "detail": "手动加仓"}
      ],
      "buy_count": 1,
      "buy_users": ["蓝鲸跃财"],
      "sell_actions": [
        {"name": "Z先生养基", "amount": "-3000", "detail": "止盈卖出"}
      ],
      "sell_count": 1,
      "sell_users": ["Z先生养基"],
      "net_signal": "neutral"
    },
    "000216": {
      "code": "000216",
      "name": "YYY基金",
      "buy_actions": [
        {"name": "蓝鲸跃财", "amount": "+10000", "detail": "大额买入"},
        {"name": "南山隐士", "amount": "+5000", "detail": "定投买入"},
        {"name": "黑夜银翼", "amount": "+3000", "detail": "手动加仓"}
      ],
      "buy_count": 3,
      "buy_users": ["蓝鲸跃财", "南山隐士", "黑夜银翼"],
      "sell_actions": [],
      "sell_count": 0,
      "sell_users": [],
      "net_signal": "strong_buy"
    }
  }
}
```

**net_signal 生成逻辑**：
- `buy_count ≥ 3` → `strong_buy`（强买入共识）
- `buy_count ≥ 2 且 sell_count = 0` → `buy`（至少 2 人买入才视为有效信号，单人可能为定投）
- `sell_count ≥ 2` → `sell`（多人卖出=撤退信号）
- `sell_count ≥ 1 且 buy_count = 0` → `weak_sell`
- `buy_count > sell_count * 2` → `buy_with_sell`（买入远大于卖出）
- `sell_count > buy_count * 2` → `sell_with_buy`（卖出远大于买入）
- 其他 → `neutral`

### 第五步：合并分析（持仓变化 + 交易流水）

结合 Step 3（持仓变化）和 Step 4（交易流水），输出综合信号：

| 信号类型 | 数据来源 | 判断标准 | 信号强度 |
|---------|---------|---------|:-------:|
| 多人新进买入 | 交易流水 ✔️ | ≥3人近期买入同一基金 | ⭐⭐⭐ 最强 |
| 持续加仓 | 交易流水 ✔️ | 同一人反复买入（含定投） | ⭐⭐ |
| 多人同时增持 | 持仓快照 ✔️ | ≥3人持仓金额同时增加 | ⭐⭐ |
| 大额买入 | 交易流水 ✔️ | 单笔 > 1万元 | ⭐⭐ |
| 大佬清仓 | 持仓快照+交易流水 ✔️ | 持仓中消失 + 交易记录有卖出 | ⚠️ 风险 |
| 多人抱团撤退 | 交易流水 ✔️ | ≥2人近期卖出同一基金 | 🔴 红色 |
| 分歧信号 | 两者结合 ✔️ | 有人买有人卖同一基金 | ❓ 需深入 |
| 持仓无变化但有交易 | 交易流水 ✔️ | 持仓金额未变但大佬在买卖（调仓换基） | ⭐⭐ |

> **持仓快照和交易流水是互补的**：快照告诉你"谁持有什么"，流水告诉你"谁在做什么"。

## 注意事项

- **交易流水 > 持仓快照**。持仓里有的基金可能买了很久（成本未知），交易流水里正在买的才是大佬当下的判断
- 每日运行一次即可，持仓数据和交易流水每日收盘后更新
- 大佬持仓是**季度公开**的（基金季报），但京东金融实盘功能提供了**实时可见**的持仓和交易流水
- 大佬买 ≠ 你该买，大佬的仓位/期限/目标可能与你不同
- 交易流水中的"买入"信号最有价值，但需要结合自己的投资体系做最终决策
- 监控报告是**参考**，不是**决策依据**
